## 結論

現状の 06 章は、nullable reference types、generics、pattern matching、extension methods、attributes、reflection、using / namespace、required / init、record、primary constructor、generic constraints、variance、ref / out / in、Nullable&lt;T&gt; を扱っており、C# の型システムと言語機能の入口としては広く押さえている。

一方で、開発時に必要な判断材料としては不足がある。特に「どの機能をいつ使うか」「使わない方がよい場面」「DTO / Domain Model / Value Object とどう組み合わせるか」「属性や reflection を実務でどう使うか」のサンプルが薄い。

## 不足している知識

### 優先度 高

- 型設計の選び方。`class`、`record`、`record struct`、`struct`、`readonly struct`、`enum` をどう選ぶか。
- `required` / `init` / constructor validation / nullable reference types の使い分け。
- DTO、設定 class、Domain Model、Value Object での record / class の選択基準。
- pattern matching を使う場面と、polymorphism や strategy に分けた方がよい場面の判断。
- extension method を作るべき場面と、通常の service / helper にすべき場面の判断。
- attributes と reflection の関係。attribute を付けただけでは動かず、読む側の framework / code が必要であること。
- reflection を避ける判断、使う場合の性能、型安全性、trimming / AOT への注意。

### 優先度 中

- nullable warning の読み方と直し方。`!` を使う前に検討すべきこと。
- `notnull`、`class`、`class?`、`struct`、`unmanaged` など generic constraints の実務的な違い。
- `required` と `SetsRequiredMembers` の入口。
- static abstract members in interfaces など、数値 generic API で見かける機能の入口。
- discriminated union がない C# で、record hierarchy や enum + data をどう扱うか。
- source generator、serializer、DI container が attributes / reflection / metadata をどう使うかの概念。

## サンプル充足性

全記事に最小コード例はあるが、実務で迷うケースの比較サンプルは不足している。

比較的充足している記事:

- `01_nullable-reference-types.md`: 要点は実務寄りに拡充済み。ただし warning 修正例は追加余地あり。
- `08_requiredとinit.md`: 基本例はある。constructor validation との比較がほしい。
- `09_recordと値の等価性.md`: record の基本例はある。DTO / Value Object / Entity の比較がほしい。
- `15_ref-out-in.md`: 通常業務コードでは避ける判断まで書けている。
- `16_NullableT.md`: `HasValue`、`??` の基本はある。

不足が目立つ記事:

- `02_generics.md`: Result&lt;T&gt;、Repository&lt;T&gt;、ApiResponse&lt;T&gt; など実務で読む generic 型の例が不足。
- `03_pattern-matching.md`: 業務ステータス分類、property pattern、list pattern などの例が不足。
- `04_extension-methods.md`: IServiceCollection 拡張、DTO mapping 拡張など実務例と乱用防止例が不足。
- `05_attributes.md`: `JsonPropertyName`、`Obsolete`、xUnit、ASP.NET Core attribute などの比較例が不足。

- `12_共変性と反変性.md`: 入口としては十分だが、interface 設計で `in` / `out` を付ける判断例が不足。
