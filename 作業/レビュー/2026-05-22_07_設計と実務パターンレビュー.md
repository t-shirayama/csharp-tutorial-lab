## 結論

現状の 07 章は、SOLID、DI、例外、ログ、設定、代表的なデザインパターン、レイヤードアーキテクチャ、Clean Architecture、ASP.NET Core の DI / Options / ILogger / global error handling / background service を扱っており、実務設計の入口としては成立している。

一方で、開発現場で「どう設計すればよいか」を判断するには不足がある。特に、前半の記事は概念説明とレビュー観点が中心で、悪い設計から改善するサンプル、責務分割の具体例、DTO / Entity / Domain Model の使い分け、transaction 境界、UseCase / Service / Repository の責務境界が薄い。

## 不足している知識

### 優先度 高

- Application Service / Domain Service / Repository / Controller の責務分担。
- DTO、Request / Response、Entity、Domain Model、ViewModel の使い分け。
- Controller や endpoint に業務ロジックが漏れる悪い例と、service / use case へ移す改善例。
- 例外、Result 型、validation error、domain error の使い分け。
- transaction 境界をどの層で管理するか。
- Repository パターンを使う場合と使わない場合の判断。09 章の記事への導線はあるが、07 章側にも設計判断が必要。
- 設計レビューで見るべきチェックリスト。依存方向、責務、命名、変更理由、テスト容易性を横断して確認する記事がほしい。

### 優先度 中

- CQRS の入口。読み取りと書き込みを分ける判断材料。
- Result pattern / Either 的な戻り値設計の入口。
- Unit of Work と transaction script の違い。
- Feature folder / vertical slice architecture の入口。
- Mapper / assembler の役割。AutoMapper を使うか手書きするかの判断。
- Value Object と Domain Model の実務例。02 章の記事への導線はあるが、07 章の設計文脈でも扱うとよい。
- 外部 API client の adapter 境界、retry / timeout / circuit breaker との接続。

## サンプル充足性

後半の記事、特に `09_DIコンテナの実装.md`、`10_IOptionsとConfiguration.md`、`11_ILoggerと構造化ログ.md`、`12_グローバルエラーハンドリング.md`、`13_IHostedServiceとバックグラウンド処理.md` はサンプルがあり、入口としては使える。

不足が目立つ記事:

- `01_SOLID原則.md`: 原則の説明だけで、違反例と改善例がない。
- `03_例外設計.md`: コード例がなく、validation / domain error / system error の実装例がない。
- `05_設定管理.md`: 設定の分類はあるが、コード例は `10_IOptionsとConfiguration.md` に寄っている。入口としては最低限でよいが、必須設定検証の流れへの導線を強めたい。
- `06_FactoryStrategyAdapter.md`: 3パターンのコード例がないため、実装イメージが弱い。
- `07_レイヤードアーキテクチャ.md`: 層ごとのコード配置例、依存方向の悪い例、改善例がない。
- `08_CleanArchitectureの入口.md`: 概念説明中心で、UseCase と interface の具体例がない。

## 追加したいサンプル

- Controller に業務ロジックが入っている悪い例と、Application Service へ移す改善例。
- `OrderService` が repository、clock、logger を使って業務処理する例。
- domain rule 違反を `DomainException` または Result で表す例。
- validation error は例外にせず、400 response にする例。
- Factory で支払い方法ごとの Strategy を選ぶ例。
- Adapter で外部 API response を自社 model へ変換する例。
- レイヤード構成の project / folder 例。
- UseCase が transaction 境界を持ち、Repository を呼ぶ例。
- 設定値と secret を分け、Options validation へつなげる例。
