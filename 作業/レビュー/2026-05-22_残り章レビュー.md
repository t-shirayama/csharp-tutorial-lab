# 残り章レビュー

## 対象

既に個別レビュー・補強済みの `00_環境構築`、`01_基礎文法`、`03_コレクションとLINQ`、`06_型システムと言語機能`、`07_設計と実務パターン`、`08_テストと品質`、`10_WebとAPI` を除く章を対象に確認した。

- `docs/05_非同期と並行処理/README.md`

## 総評

残り章も、初学者が C# / .NET の実務開発に入るための主要知識はおおむね網羅されている。特に `02_オブジェクト指向`、`04_標準ライブラリ`、`05_非同期と並行処理`、`13_ランタイムと高度なCSharp` は、記事ごとにコード例、コードの読み方、よくあるミスが揃っており、サンプル充足度は高い。

一方で、実務で即戦力という観点では `09_データアクセス`、`11_ツールと運用`、`12_実践プロジェクト` に追加余地がある。特に DB 性能診断、Raw SQL、複数 DbContext、secret 管理の運用例、デプロイ失敗時の診断、実践プロジェクトの step-by-step 実装ガイドは、開発現場で詰まりやすい。

## 不足している点

1. `09_データアクセス` の性能診断が、実務の調査手順としてはまだ薄い。
   - SQL ログの読み方、`ToQueryString()`、slow query、index、実行計画、`AsNoTracking`、projection の判断例をまとめたい。

2. `09_データアクセス` に Raw SQL と DB 製品差分の記事がない。
   - `FromSql`、`ExecuteSql`、Dapper 併用、SQL Server / PostgreSQL / SQLite の型・接続文字列・migration 差分を扱うと実務に近づく。

3. `09_データアクセス` に複数 DbContext や read/write 分離の設計例がない。
   - bounded context、readonly query context、transaction 境界、migration 分割の判断があると中規模以上に対応しやすい。

4. `11_ツールと運用` の secret 管理は入口としてはあるが、運用例が薄い。
   - user-secrets、environment variables、GitHub Actions secrets、Azure Key Vault などの使い分けと注意点を補強したい。

5. `11_ツールと運用` にデプロイ失敗時の診断記事がない。
   - GitHub Actions log、container 起動失敗、設定漏れ、rollback、health check の確認を扱うと運用に強くなる。

6. `12_実践プロジェクト` は、課題としては良いが手順の粒度が粗い。
   - TODO Web API、DB 付き CRUD、認証付きメモ API に、実装順、主要ファイル、確認コマンド、テスト追加、PR 前確認までの具体手順がほしい。

7. `13_ランタイムと高度なCSharp` は入口として十分だが、plugin 実装は浅い。
   - AssemblyLoadContext を使った plugin discovery、unload、依存 assembly 解決、よくある load error を追加すると実務調査に使いやすい。

## 判定

- 網羅性: 残り章も基本実務に必要な知識はおおむね網羅されている。
- サンプル: 02、04、05、13 は充足。09、11 は入口サンプルはあるが、実務の調査・運用手順は追加余地あり。12 は題材は充足しているが、中間実装サンプルが不足。
- 優先対応: DB 性能診断、実践プロジェクト step-by-step 化、デプロイ失敗時診断を先に補強する。
