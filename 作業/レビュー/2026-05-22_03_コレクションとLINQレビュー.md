## 結論

現状の 03 章は、配列、List、Dictionary、IEnumerable、LINQ、遅延評価、GroupBy、Join、ToList、HashSet、Queue、Stack、不変コレクション、Index / Range を扱っており、初学者が C# の collection / LINQ の全体像をつかむ入口としては成立している。

一方で、実務開発で頻出する API 選択、更新中の列挙、取得系 LINQ の使い分け、ソート、重複排除、ページング、比較器、読み取り専用公開、スレッド安全性との境界などは不足している。サンプルも各記事 1〜2 個の最小例が中心で、開発時に迷いやすい失敗例と改善例がまだ薄い。

## 不足している知識

### 優先度 高

- `List<T>`、配列、`IReadOnlyList<T>`、`IEnumerable<T>` の使い分け。
- collection を外部公開するときの mutable / readonly / immutable の違い。
- `Single`、`SingleOrDefault`、`First`、`FirstOrDefault`、`ElementAtOrDefault` の使い分け。
- `OrderBy`、`ThenBy`、`Distinct`、`Take`、`Skip`、`Chunk` など、一覧表示や API response で頻出する LINQ。
- `Dictionary` と `Lookup` / `ToDictionary` / `ToLookup` の使い分け。
- `foreach` 中に collection を変更した場合の例外と、安全な更新方法。
- `List<T>.Contains` と `HashSet<T>.Contains` の性能差、比較器 `IEqualityComparer<T>` の考え方。

### 優先度 中

- `Collection<T>`、`ReadOnlyCollection<T>`、`IReadOnlyCollection<T>` の位置付け。
- `Span<T>` / `Memory<T>` との境界は 13 章への導線で十分だが、Index / Range 記事から参照を強める余地がある。
- `ConcurrentDictionary` や thread-safe collection は 05 章との境界として案内が必要。
- `OfType`、`Cast`、`Zip`、`Aggregate`、`DefaultIfEmpty` など、必要時に調べられる入口。
