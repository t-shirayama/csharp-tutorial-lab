# ファクトチェック

## 目的

各 Markdown ファイルについて、内容が公式情報や一次情報と矛盾していないか確認し、確認日時を残します。

## 記録単位

ファクトチェックは **ファイルごと** に行います。1つの記事を確認したら、[台帳.md](台帳.md) に1行追加または更新し、必要に応じて `記録/` に詳細記録を作ります。

## 必ず記録する項目

- 対象ファイル
- 確認日時
- 確認者
- 確認した観点
- 根拠リンク
- 結果
- 次回確認目安

## 結果の種類

| 結果 | 意味 |
| --- | --- |
| OK | 公式情報と矛盾なし |
| 要修正 | 本文修正が必要 |
| 要確認 | 根拠不足、または追加調査が必要 |
| 対象外 | テンプレート、作業メモなど fact の確認対象外 |

## 運用

1. 対象ファイルを1つ選ぶ。
2. 公式ドキュメント、Microsoft Learn、GitHub 公式リポジトリなど一次情報で確認する。
3. [台帳.md](台帳.md) に確認日時と結果を記録する。
4. 修正が必要なら `作業/レビュー/` にレビュー記録を作る。
5. 本文修正が完了したらレビュー記録を `作業/アーカイブ/` に移す。
