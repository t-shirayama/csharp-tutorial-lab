# 設定管理

## 目的

環境ごとに変わる値をコードから分離して管理する考え方を理解します。

## 要点

- 設定値はコードに直書きしません。
- 環境変数、appsettings、User Secrets、Key Vault などを使い分けます。
- 秘密情報は Git に入れません。

## 実務での使い方

接続文字列、外部 API URL、タイムアウト、機能フラグなどは設定化します。起動時に必須設定を検証すると、実行中の不可解な失敗を減らせます。

`IConfiguration` から型付き設定へ bind し、`IOptions<T>` と validation で扱う実装は [10_IOptionsとConfiguration.md](10_IOptionsとConfiguration.md) で扱います。

## よくあるミス

- 開発用接続文字列をコミットする。
- 設定名が統一されていない。
- 設定がない場合に null のまま処理が進む。

## レビュー観点

- 設定値がコードに直書きされていないか。
- 必須設定を起動時に検証しているか。
- 秘密情報と非秘密情報を分けているか。
- 環境差分が appsettings、環境変数、User Secrets などで説明できるか。
