# 非同期処理のよくある落とし穴

## 目的

async / await を使うときに起きやすい問題を整理し、レビュー時に見つけられるようにします。

## 要点

- `.Result` と `.Wait()` はデッドロックやスレッド枯渇の原因になります。
- `async void` はイベントハンドラ以外では避けます。
- キャンセルとタイムアウトを考慮します。
- 並行数を制限しない大量リクエストは危険です。

## 悪い例

```csharp
// この例では「非同期処理のよくある落とし穴」の要点を最小のコードで確認します。
var text = httpClient.GetStringAsync("https://example.com").Result;
```

## よい例

```csharp
// この例では「非同期処理のよくある落とし穴」の要点を最小のコードで確認します。
var text = await httpClient.GetStringAsync("https://example.com", cancellationToken);
```

## 実務でのレビュー観点

- async メソッドが呼び出し元まで `await` されているか。
- `CancellationToken` が下位処理まで渡されているか。
- 外部 API 呼び出しにタイムアウトや失敗時の扱いがあるか。
- 大量並行実行に制限があるか。

## コードの読み方

このコード例は「非同期処理のよくある落とし穴」の基本形を確認するためのものです。上から順に、値や object を用意し、C# の構文や .NET API を使い、最後に結果を確認します。まず入力、処理、出力の 3 つに分けて読むと、初学者でも流れを追いやすくなります。

## 練習問題

1. `.Result` を使ったコードを `await` に直す。
2. `CancellationToken` を引数に追加する。
3. `Task.WhenAll` の同時実行数を制限する方法を調べる。
