# タイムアウトとリトライ

## 目的

外部 API や DB などの一時的な失敗に対して、タイムアウトとリトライを設計できるようにします。

## 前提

- [CancellationToken](03_CancellationToken.md) を読んでいる
- [HttpClientFactory](../04_標準ライブラリ/09_HttpClientFactory.md) を読んでいる

## 要点

- timeout は待ち続けないための上限です。
- retry は一時的な失敗だけに使います。
- retry してはいけない操作や、重複実行の危険を考えます。
- cancellation と timeout の責務を分けます。

## コード例

```csharp
// 5秒で timeout する token を作ります。
using var timeout = new CancellationTokenSource(TimeSpan.FromSeconds(5));

// 呼び出し元の cancellationToken と timeout token をまとめます。
using var linked = CancellationTokenSource.CreateLinkedTokenSource(
    timeout.Token,
    cancellationToken);

// 外部 API 呼び出しに linked token を渡し、キャンセルと timeout の両方に反応させます。
await httpClient.GetAsync("/health", linked.Token);
```

## コードの読み方

`timeout.Token` は「5秒経ったら止める」ための token です。呼び出し元から渡された `cancellationToken` と結合することで、ユーザー操作によるキャンセルと timeout の両方を同じ API 呼び出しに伝えられます。

## 実務での使い方

外部 API 呼び出し、DB 接続、キュー処理で使います。POST の retry は重複登録につながることがあるため、冪等性や idempotency key を検討します。

## よくあるミス

- timeout なしで外部 API を待ち続ける。
- すべての例外を retry する。
- retry 間隔を空けずに連続実行する。
- キャンセル要求を retry で握りつぶす。
