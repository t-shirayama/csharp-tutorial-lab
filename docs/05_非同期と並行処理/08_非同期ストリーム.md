# 非同期ストリーム

## 目的

`IAsyncEnumerable<T>` と `await foreach` で、非同期に届くデータを順次処理できるようにします。

## 前提

- [async / await](02_async-await.md) を読んでいる
- [CancellationToken](03_CancellationToken.md) を読んでいる

## 要点

- `IAsyncEnumerable<T>` は非同期に複数の値を返す stream です。
- `await foreach` で1件ずつ処理します。
- 大量データを一度に `List<T>` にしない選択肢になります。

## コード例

```csharp
// この例では「非同期ストリーム」の要点を最小のコードで確認します。
public async IAsyncEnumerable<int> GenerateAsync()
{
    for (var i = 0; i < 3; i++)
    {
        // 値が届くまでの非同期待ちを表現しています。
        await Task.Delay(100);

        // yield return で1件ずつ呼び出し側へ返します。
        yield return i;
    }
}

// await foreach は、非同期に届く値を順番に処理します。
await foreach (var value in GenerateAsync())
{
    Console.WriteLine(value);
}
```

## コードの読み方

`GenerateAsync` は `Task<List<int>>` ではなく `IAsyncEnumerable<int>` を返しています。すべての値が揃うまで待つのではなく、値が1件ずつ届くたびに `await foreach` の中で処理できます。

## 実務での使い方

ページングされた API、ファイルの逐次読み込み、DB 結果の stream 処理などで使います。途中でキャンセルできるように `CancellationToken` も設計します。

## よくあるミス

- 逐次処理でよいのに全件 `ToListAsync()` する。
- 例外発生時の途中状態を考えない。
- stream を複数回列挙できる前提で扱う。
