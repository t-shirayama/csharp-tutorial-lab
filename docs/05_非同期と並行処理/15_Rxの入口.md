# Rx の入口

## 目的

Reactive Extensions の基本概念である `IObservable<T>` と `IObserver<T>` を理解します。

## 前提

- [非同期ストリーム](08_非同期ストリーム.md) を読んでいる
- [delegate と event](../02_オブジェクト指向/13_delegateとevent.md) を読んでいる

## 要点

- Rx は時間を伴う event stream を扱う library です。
- `IObservable<T>` は値を通知する source です。
- `IObserver<T>` は通知を受け取る subscriber です。
- LINQ に似た演算子で stream を変換できます。

## パッケージ

```powershell
dotnet add package System.Reactive
```

## コード例

```csharp
// この例では「Rx の入口」の要点を最小のコードで確認します。
using System.Reactive.Linq;

var subscription = Observable.Interval(TimeSpan.FromSeconds(1))
    .Take(3)
    .Subscribe(value => Console.WriteLine(value));

await Task.Delay(TimeSpan.FromSeconds(4));
subscription.Dispose();
```

## コードの読み方

`Observable.Interval` は一定間隔で値を発行します。`Take(3)` は最初の 3 件だけに制限します。`Subscribe` は購読を開始し、戻り値の `IDisposable` を破棄すると購読を解除します。

## 実務での使い方

UI event、監視、センサー、ストリーム処理、時間を伴う event の合成で使います。Web API の通常処理では必須ではないため、必要な場面が見えてから深掘りします。

## よくあるミス

- subscription を dispose せず、通知が残り続ける。
- `IEnumerable<T>`、`IAsyncEnumerable<T>`、`IObservable<T>` の違いを混同する。
- scheduler を意識せず、UI thread や background thread の問題を起こす。
