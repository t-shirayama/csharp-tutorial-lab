# 入力と出力

## 目的

コンソールアプリでユーザー入力を受け取り、結果を表示する基本を理解します。

## 前提

- [条件分岐](03_条件分岐.md) を読んでいる
- [メソッド](05_メソッド.md) を読んでいる

## 要点

- `Console.ReadLine()` は文字列または `null` を返します。
- 数値に変換するときは `int.TryParse` を使うと失敗を扱いやすいです。
- 入力、変換、計算、出力を分けるとテストしやすくなります。

## コード例

```csharp
// この例では「入力と出力」の要点を最小のコードで確認します。
Console.Write("Age: ");
var input = Console.ReadLine();

if (!int.TryParse(input, out var age))
{
    Console.WriteLine("数値を入力してください。");
    return;
}

Console.WriteLine($"来年は {age + 1} 歳です。");
```

## コードの読み方

この例は、入力、変換、分岐、出力の順に読めます。`Console.ReadLine()` は `string?` を返すため、入力がない可能性があります。`int.TryParse(input, out var age)` は、文字列を整数に変換できるかを確認し、成功した場合だけ `age` を使います。失敗したら `return` で早めに終了するため、後続の処理は正しい値だけを前提にできます。

## 実務での使い方

CLI ツールや学習用アプリでは、入力値の検証とエラーメッセージが重要です。業務ロジックは `Console` から切り離すと、後で Web API やテストへ移しやすくなります。

入力を整える方法は [文字列操作](14_文字列操作.md)、数値や日付へ変換する方法は [型変換と Parse](15_型変換とParse.md) と [日付と時刻の基本](16_日付と時刻の基本.md) で扱います。

## よくあるミス

- `ReadLine()` の戻り値が必ず文字列だと思う。
- `int.Parse` で変換失敗時に例外終了する。
- 入力処理と計算処理を1つのメソッドに詰め込む。
