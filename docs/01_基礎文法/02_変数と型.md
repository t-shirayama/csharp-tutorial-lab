# 変数と型

## 目的

C# で値を扱うための基本である、変数、型、代入、型推論を理解します。

## 前提

- [C# のプログラム構造](01_プログラム構造.md) を読んでいる
- コンソールアプリを作成して `dotnet run` できる

## 要点

- 変数は値に名前を付ける仕組みです。名前は後から読む人にとっての説明にもなるため、`x` や `data` のような曖昧な名前より、`customerName`、`totalPrice`、`isActive` のように意味が伝わる名前を優先します。
- C# は静的型付け言語なので、変数には必ず型があります。型があることで、文字列を数値として計算する、null の可能性を見落とす、といったミスをコンパイル時に見つけやすくなります。
- `var` は型がない書き方ではなく、右辺から型を推論する書き方です。`var age = 30;` は `int age = 30;` と同じ型になり、実行中に別の型へ変わるわけではありません。
- 実務では、右辺を見れば型が明らかな `new User()` や LINQ の途中結果では `var` が読みやすく、金額、日時、外部 API の戻り値など意図を明確にしたい場面では明示的な型が役立ちます。
- 数値型は用途で選びます。件数や年齢は `int`、大きな ID や集計値は `long`、金額は丸め誤差を避けるため `decimal` を使うのが基本です。
- null を許すかどうかは設計上の重要な判断です。`string` と `string?` を使い分けることで、「必ず値がある」のか「未設定があり得る」のかをコード上で表現できます。

## 基本例

```csharp
string name = "Sato";      // string は文字列を表す
int age = 30;              // int は整数を表す
decimal price = 1200.5m;   // decimal は金額などの10進数に向く
bool isActive = true;      // bool は true / false の2択を表す

Console.WriteLine($"{name}: {age}歳 / {price}円 / active={isActive}");
```

## よく使う型

| 型 | 用途 | 例 |
| --- | --- | --- |
| `string` | 文字列 | `"hello"` |
| `char` | 1文字 | `'A'` |
| `int` | 整数 | `100` |
| `long` | 大きな整数 | `10000000000L` |
| `decimal` | 金額などの10進数 | `1200.5m` |
| `double` | 小数、計測値 | `3.14` |
| `bool` | 真偽値 | `true` / `false` |
| `DateTime` | 日時 | `DateTime.Now` |

## var の使い方

```csharp
var name = "Sato";             // 右辺が文字列なので string と推論される
var age = 30;                  // 右辺が整数なので int と推論される
var createdAt = DateTime.Now;  // 戻り値の型が DateTime なので DateTime と推論される

Console.WriteLine(name);
Console.WriteLine(age);
Console.WriteLine(createdAt);
```

`var` は便利ですが、右辺を見ても型が分かりにくい場合は明示的な型を書いた方が読みやすくなります。

```csharp
Dictionary<string, int> scores = new();  // 科目名 string と点数 int の対応表
scores["math"] = 80;                     // "math" というキーに 80 点を保存する
```

## null と nullable

参照型の値が存在しない状態を `null` と呼びます。

```csharp
string? optionalName = null;  // ? が付くので null を入れてもよい

if (optionalName is null)
{
    Console.WriteLine("名前は未設定です。");
}
```

`string?` の `?` は、null になる可能性があることを示します。実務では null の扱いがバグの原因になりやすいため、nullable reference types を有効にして意図を明確にします。

## よくあるミス

- `int` に小数を代入しようとする。
- `decimal` のリテラルに `m` を付け忘れる。
- `var value;` のように、右辺なしで `var` を使おうとする。
- null になる可能性がある値をチェックせずに使う。

## コードの読み方

基本例では、`name`、`age`、`price`、`isActive` がそれぞれ違う型の値を持っています。`decimal price = 1200.5m;` の `m` は decimal literal を表し、金額を `double` ではなく 10 進数として扱うために必要です。最後の `Console.WriteLine` は string interpolation で複数の値を表示用文字列に埋め込んでいます。

## 実務での使い方

実務コードでは、変数名と型の選び方が読みやすさに直結します。金額には `double` ではなく `decimal`、ID や件数には `int` や `long`、日時には `DateTimeOffset` を検討するなど、値の意味に合う型を選びます。

文字列の整形は [文字列操作](14_文字列操作.md)、外部入力の変換は [型変換と Parse](15_型変換とParse.md)、日付や時刻は [日付と時刻の基本](16_日付と時刻の基本.md) で扱います。

## 練習問題

1. 商品名、単価、数量を変数に入れて、合計金額を表示する。
2. `var` を使った場合と明示的な型を書いた場合で、読みやすさを比べる。
3. null になる可能性がある `string?` を定義し、null のときだけメッセージを表示する。
