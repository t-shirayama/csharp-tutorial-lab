# デバッグの基本

## 目的

エラーや想定外の結果を、再現、観察、仮説、確認の流れで切り分けられるようにします。

## 前提

- [C# のプログラム構造](01_プログラム構造.md) を読んでいる
- VS Code で C# project を開ける

## 要点

- まず「期待した結果」と「実際の結果」を分けます。
- エラーは最初の error と stack trace を読みます。
- 小さな入力で再現させると原因を絞りやすくなります。
- `Console.WriteLine`、breakpoint、debugger を使い分けます。

## 確認観点

```text
1. どのコマンドで起きたか
2. どの入力で再現するか
3. 期待値と実際の値は何か
4. 最初のエラー文は何か
5. 直前に変更したコードは何か
```

## 実務での使い方

レビューや質問では、再現手順、入力、期待値、実際の結果をセットで伝えます。原因が分からない段階でも、観察事実を整理できる人は調査が速くなります。

## よくあるミス

- エラー全文を読まずに最後の行だけ見る。
- 複数箇所を同時に変えて原因が分からなくなる。
- 再現条件を記録しない。
- debugger で値を見ずに推測だけで直す。
