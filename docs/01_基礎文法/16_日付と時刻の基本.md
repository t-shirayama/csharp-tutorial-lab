# 日付と時刻の基本

## 目的

C# で日付、時刻、期間を扱うための基本を理解します。

## 前提

- [変数と型](02_変数と型.md) を読んでいる
- [型変換と Parse](15_型変換とParse.md) を読んでいる

## 要点

- `DateTime` は日付と時刻を表します。`DateTime.Today` は今日の日付、`DateTime.Now` はローカル時刻の現在日時を返します。
- `DateTimeOffset` は日時に offset を含めて表します。外部 API、ログ、保存データでは、時差やタイムゾーンの誤解を減らすため `DateTimeOffset` が使われることがあります。
- 現在時刻を使う処理はテストしにくくなりやすいです。基礎段階では `DateTimeOffset.UtcNow` を知り、実務では `TimeProvider` などで時刻取得を差し替える設計へ進みます。
- 日付の加算や減算には `AddDays`、`AddMonths`、`AddHours` などを使います。日数や時間差は `TimeSpan` として扱えます。
- 表示形式は `ToString("yyyy-MM-dd")` のように指定できます。ただし保存や API 連携では、表示用の文字列と内部の日時型を混同しないようにします。
- ユーザー入力の日付は `DateTime.TryParse` で変換できますが、入力形式が曖昧だと誤解が起きます。実務では入力形式を画面や仕様で明確にします。

## 現在日時を取得する

```csharp
var localNow = DateTime.Now;
var utcNow = DateTimeOffset.UtcNow;

Console.WriteLine($"ローカル時刻: {localNow}");
Console.WriteLine($"UTC: {utcNow}");
```

## 日付を計算する

```csharp
var orderedAt = DateTime.Today;

// 3営業日ではなく単純に3日後です。営業日計算は別の業務ルールが必要です。
var estimatedShippingDate = orderedAt.AddDays(3);

Console.WriteLine($"注文日: {orderedAt:yyyy-MM-dd}");
Console.WriteLine($"出荷予定日: {estimatedShippingDate:yyyy-MM-dd}");
```

## 日付入力を変換する

```csharp
Console.Write("期限を入力してください yyyy-MM-dd: ");
var input = Console.ReadLine();

if (!DateTime.TryParse(input, out var dueDate))
{
    Console.WriteLine("日付として解釈できません。");
    return;
}

if (dueDate.Date < DateTime.Today)
{
    Console.WriteLine("期限は今日以降の日付にしてください。");
    return;
}

Console.WriteLine($"期限: {dueDate:yyyy-MM-dd}");
```

## TimeSpan で期間を扱う

```csharp
var startedAt = DateTimeOffset.UtcNow;
var finishedAt = startedAt.AddSeconds(90);

TimeSpan elapsed = finishedAt - startedAt;

Console.WriteLine($"処理時間: {elapsed.TotalSeconds}秒");
```

## コードの読み方

日付入力の例では、文字列を `DateTime` に変換してから、業務ルールとして「今日以降か」を確認しています。`TryParse` は形式チェック、`dueDate.Date < DateTime.Today` は業務チェックです。表示では `:yyyy-MM-dd` を指定し、画面に出す形式を固定しています。

## 実務での使い方

期限、予約日時、ログ時刻、有効期限、timeout などで日付と時刻を扱います。保存や API では UTC や offset を含む形式を検討し、画面表示ではユーザーの地域や要件に合わせて整形します。業務日、営業日、締め日、月末処理は単純な `AddDays` だけでは足りないことが多いです。

## よくあるミス

- `DateTime.Now` をどこでも直接呼び、テストで現在時刻を固定できなくなる。
- UTC とローカル時刻を混ぜて比較する。
- 日付表示用の文字列を内部処理にも使い回す。
- 営業日計算を単純な `AddDays` で済ませてしまう。
