# 小さなコンソールアプリ実践

## 目的

基礎文法を組み合わせて、入力、検証、計算、出力を持つ小さなコンソールアプリを作れるようにします。

## 作るもの

商品名、単価、数量を入力し、税込合計を表示する簡単な注文計算アプリを作ります。

## 学習ポイント

- `Console.ReadLine()` で入力を受け取る。
- `Trim()` と `IsNullOrWhiteSpace` で文字列を検証する。
- `decimal.TryParse` と `int.TryParse` で型変換する。
- ガード句で不正な入力を早めに返す。
- 計算処理をメソッドに分ける。
- 表示用の文字列を組み立てる。

## コード例

```csharp
const decimal TaxRate = 0.10m;

Console.Write("商品名: ");
var productName = ReadRequiredText(Console.ReadLine(), "商品名");
if (productName is null)
{
    return;
}

Console.Write("単価: ");
var unitPrice = ReadDecimal(Console.ReadLine(), "単価");
if (unitPrice is null)
{
    return;
}

Console.Write("数量: ");
var quantity = ReadInt(Console.ReadLine(), "数量");
if (quantity is null)
{
    return;
}

var subtotal = CalculateSubtotal(unitPrice.Value, quantity.Value);
var total = CalculateTotalWithTax(subtotal, TaxRate);

Console.WriteLine("--- 注文内容 ---");
Console.WriteLine($"商品名: {productName}");
Console.WriteLine($"小計: {subtotal}円");
Console.WriteLine($"税込合計: {total}円");

static string? ReadRequiredText(string? input, string fieldName)
{
    var value = (input ?? string.Empty).Trim();

    if (string.IsNullOrWhiteSpace(value))
    {
        Console.WriteLine($"{fieldName}は必須です。");
        return null;
    }

    return value;
}

static decimal? ReadDecimal(string? input, string fieldName)
{
    if (!decimal.TryParse(input, out var value))
    {
        Console.WriteLine($"{fieldName}は数値で入力してください。");
        return null;
    }

    if (value < 0)
    {
        Console.WriteLine($"{fieldName}は0以上にしてください。");
        return null;
    }

    return value;
}

static int? ReadInt(string? input, string fieldName)
{
    if (!int.TryParse(input, out var value))
    {
        Console.WriteLine($"{fieldName}は整数で入力してください。");
        return null;
    }

    if (value <= 0)
    {
        Console.WriteLine($"{fieldName}は1以上にしてください。");
        return null;
    }

    return value;
}

static decimal CalculateSubtotal(decimal unitPrice, int quantity)
{
    return unitPrice * quantity;
}

static decimal CalculateTotalWithTax(decimal subtotal, decimal taxRate)
{
    return subtotal * (1 + taxRate);
}
```

## コードの読み方

最初に定数 `TaxRate` を決め、商品名、単価、数量を順に読み取ります。各入力は専用メソッドで検証し、失敗したら `null` を返して処理を終了します。計算は `CalculateSubtotal` と `CalculateTotalWithTax` に分けているため、入力処理と計算処理が混ざりません。

`ReadDecimal` と `ReadInt` は、変換できるかどうかと、業務上使える値かどうかを分けて確認しています。たとえば単価は `0` でも許可していますが、数量は `1` 以上にしています。この違いが業務ルールです。

## 完了条件

- 商品名、単価、数量を入力できる。
- 商品名が空ならエラーメッセージを表示して終了する。
- 単価が数値でない、または 0 未満ならエラーメッセージを表示して終了する。
- 数量が整数でない、または 1 未満ならエラーメッセージを表示して終了する。
- 正しい入力なら小計と税込合計を表示する。

## 実務での使い方

実務の画面や API でも、基本は同じです。外から入った値を検証し、型変換し、業務処理へ渡し、結果を返します。コンソールアプリでは `Console.WriteLine` で返していますが、Web API なら response、desktop app なら画面表示、batch ならログやファイル出力に置き換わります。

## よくあるミス

- 入力、変換、計算、出力をすべて1つの長い処理に詰め込む。
- `TryParse` の失敗時にも計算を続ける。
- 単価と数量の validation ルールを同じにしてしまう。
- 税率を magic number として複数箇所に直接書く。
