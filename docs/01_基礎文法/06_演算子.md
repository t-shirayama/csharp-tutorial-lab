# 演算子

## 目的

C# で値を計算、比較、結合、判定するための基本的な演算子を理解します。

## 前提

- [変数と型](02_変数と型.md) を読んでいる
- [条件分岐](03_条件分岐.md) を読んでいる

## 要点

- 演算子は値に対する操作を表します。
- 算術、比較、論理、代入、null 関連の演算子は特によく使います。
- 実務では、短く書けることよりも誤読されにくいことを優先します。

## 算術演算子

```csharp
// この例では「演算子」の要点を最小のコードで確認します。
var sum = 10 + 3;        // 13
var difference = 10 - 3; // 7
var product = 10 * 3;    // 30
var quotient = 10 / 3;   // 3
var remainder = 10 % 3;  // 1

Console.WriteLine(sum);
Console.WriteLine(difference);
Console.WriteLine(product);
Console.WriteLine(quotient);
Console.WriteLine(remainder);
```

`int` 同士の割り算は整数になります。小数として扱いたい場合は `decimal` や `double` を使います。

```csharp
// この例では「演算子」の要点を最小のコードで確認します。
var intResult = 10 / 3;
var decimalResult = 10m / 3m;

Console.WriteLine(intResult);     // 3
Console.WriteLine(decimalResult); // 3.3333333333333333333333333333
```

## 比較演算子

```csharp
// この例では「演算子」の要点を最小のコードで確認します。
var age = 20;

Console.WriteLine(age == 20); // true
Console.WriteLine(age != 20); // false
Console.WriteLine(age >= 18); // true
Console.WriteLine(age < 18);  // false
```

比較演算子の結果は `bool` です。`if` や `while` の条件でよく使います。

## 論理演算子

```csharp
// この例では「演算子」の要点を最小のコードで確認します。
var age = 20;
var hasTicket = true;

if (age >= 18 && hasTicket)
{
    Console.WriteLine("入場できます。");
}

if (age < 18 || !hasTicket)
{
    Console.WriteLine("確認が必要です。");
}
```

- `&&` は両方が `true` のとき `true` です。
- `||` はどちらかが `true` のとき `true` です。
- `!` は真偽を反転します。

## 代入演算子

```csharp
// この例では「演算子」の要点を最小のコードで確認します。
var count = 0;

count += 1;
count++;
count -= 1;

Console.WriteLine(count);
```

`+=` や `-=` は、現在の値を使って更新する短い書き方です。

## null 関連の演算子

```csharp
// この例では「演算子」の要点を最小のコードで確認します。
string? name = null;

var displayName = name ?? "未設定";
Console.WriteLine(displayName);
```

`??` は左辺が null でなければ左辺を使い、null なら右辺を使います。

```csharp
// この例では「演算子」の要点を最小のコードで確認します。
string? text = "hello";
var length = text?.Length;

Console.WriteLine(length);
```

`?.` は左辺が null の場合に例外を出さず、結果を null にします。

## よくあるミス

- `=` と `==` を混同する。
- `int / int` の結果が小数になると思い込む。
- `&&` と `||` の優先順位を意識せず、条件を誤読する。
- `!` を重ねすぎて、条件の意味が読みにくくなる。
- null 合体演算子 `??` で、空文字 `""` も置き換わると誤解する。

## コードの読み方

このコード例は「演算子」の基本形を確認するためのものです。上から順に、値や object を用意し、C# の構文や .NET API を使い、最後に結果を確認します。まず入力、処理、出力の 3 つに分けて読むと、初学者でも流れを追いやすくなります。

## 実務での使い方

実務では、条件式が仕様そのものを表すことがあります。複雑な条件は、意味のある変数名やメソッド名に分けるとレビューしやすくなります。

```csharp
// この例では「演算子」の要点を最小のコードで確認します。
var isAdult = age >= 18;
var canEnter = isAdult && hasTicket;

if (canEnter)
{
    Console.WriteLine("入場できます。");
}
```
