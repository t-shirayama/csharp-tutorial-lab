# 繰り返し

## 目的

複数の値や同じ処理を扱うための `for`、`foreach`、`while` を理解します。

## 前提

- [変数と型](02_変数と型.md) を読んでいる
- [条件分岐](03_条件分岐.md) を読んでいる

## 要点

- 回数が決まっている繰り返しには `for` を使います。
- コレクションの要素を順に処理するには `foreach` を使います。
- 条件が満たされる間だけ続ける処理には `while` を使います。
- 実務では、読みやすさと終了条件の明確さが重要です。

## for

```csharp
// この例では「繰り返し」の要点を最小のコードで確認します。
for (var index = 0; index < 5; index++)
{
    Console.WriteLine(index);
}
```

`for` は、カウンターの初期値、継続条件、更新処理を1行にまとめます。

## foreach

```csharp
// この例では「繰り返し」の要点を最小のコードで確認します。
var names = new[] { "Sato", "Suzuki", "Tanaka" };

foreach (var name in names)
{
    Console.WriteLine(name);
}
```

配列やリストの中身を順番に処理するなら、`foreach` が読みやすいことが多いです。

## while

```csharp
// この例では「繰り返し」の要点を最小のコードで確認します。
var count = 0;

while (count < 3)
{
    Console.WriteLine(count);
    count++;
}
```

`while` は条件が `true` の間だけ処理を繰り返します。

## break と continue

```csharp
// この例では「繰り返し」の要点を最小のコードで確認します。
for (var number = 1; number <= 10; number++)
{
    if (number == 5)
    {
        continue;
    }

    if (number == 8)
    {
        break;
    }

    Console.WriteLine(number);
}
```

- `continue` は現在の回だけスキップします。
- `break` は繰り返し全体を終了します。

## よくあるミス

- 終了条件を間違えて無限ループになる。
- 配列の範囲外にアクセスする。
- `foreach` の中でコレクション自体を変更して例外になる。
- ネストが深くなり、処理の意図が読みにくくなる。

## コードの読み方

このコード例は「繰り返し」の基本形を確認するためのものです。上から順に、値や object を用意し、C# の構文や .NET API を使い、最後に結果を確認します。まず入力、処理、出力の 3 つに分けて読むと、初学者でも流れを追いやすくなります。

## 実務での使い方

実務では、ファイルの行、DB から取得したデータ、API のレスポンス、画面入力の一覧などを処理するために繰り返しを使います。単純な変換や抽出は LINQ の方が読みやすい場合もあります。
