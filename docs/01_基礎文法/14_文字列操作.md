# 文字列操作

## 目的

入力値や表示文字列を扱うために、C# の基本的な文字列操作を理解します。

## 前提

- [変数と型](02_変数と型.md) を読んでいる
- [条件分岐](03_条件分岐.md) を読んでいる
- [入力と出力](08_入力と出力.md) を読んでいる

## 要点

- `string` は文字列を表す型です。ユーザー入力、ファイル内容、API の message、画面表示など、多くの場面で使います。
- `Console.ReadLine()` で受け取った文字列は、前後に空白が入ることがあります。入力値として使う前に `Trim()` で整えると、余分な空白による判定ミスを減らせます。
- 空文字 `""`、空白だけの文字列、`null` は区別されます。入力必須チェックでは `string.IsNullOrWhiteSpace` を使うと、未入力と空白だけの入力をまとめて扱えます。
- 文字列を埋め込んで表示するときは string interpolation を使うと読みやすくなります。`$"合計: {total}円"` のように、表示したい値をその場で埋め込めます。
- 文字列比較は、大文字小文字や文化圏の影響を受けることがあります。ID、command、code のような機械的な比較では `StringComparison.OrdinalIgnoreCase` などを明示すると意図が伝わります。
- `Contains`、`StartsWith`、`EndsWith`、`Split` はよく使います。ただし複雑な parsing や CSV 処理を手書きで頑張りすぎず、標準 API や専用 library を検討します。

## 空白を取り除く

```csharp
Console.Write("名前を入力してください: ");
var input = Console.ReadLine();

// ReadLine は null を返す可能性があるため、?? で空文字に置き換えます。
var name = (input ?? string.Empty).Trim();

if (string.IsNullOrWhiteSpace(name))
{
    Console.WriteLine("名前は必須です。");
    return;
}

Console.WriteLine($"こんにちは、{name}さん。");
```

## 文字列を検索する

```csharp
var fileName = "report-2026.csv";

// 拡張子や prefix のような機械的な判定では、比較ルールを明示します。
var isCsv = fileName.EndsWith(".csv", StringComparison.OrdinalIgnoreCase);
var isReport = fileName.StartsWith("report", StringComparison.OrdinalIgnoreCase);

if (isCsv && isReport)
{
    Console.WriteLine("レポート CSV です。");
}
```

## Split で分割する

```csharp
var line = "apple,banana,orange";
var items = line.Split(',');

foreach (var item in items)
{
    // 分割後の値にも空白が残ることがあるため、必要に応じて Trim します。
    Console.WriteLine(item.Trim());
}
```

## コードの読み方

最初の例では、入力、整形、検証、出力の順に処理しています。`input ?? string.Empty` は null 対策、`Trim()` は前後の空白対策、`IsNullOrWhiteSpace` は未入力対策です。実務ではこの順番を崩すと、空白だけの入力を有効扱いしたり、null で例外になったりします。

## 実務での使い方

実務では、文字列操作は validation、検索条件、ログ message、CSV や設定値の読み取りで頻繁に出てきます。文字列をそのまま比較する前に、「空白は許すか」「大文字小文字を区別するか」「null は来るか」を確認します。

## よくあるミス

- `Trim()` した結果を変数に戻さず、元の文字列が変わったと思い込む。
- `IsNullOrEmpty` だけを使い、空白だけの入力を通してしまう。
- command や code の比較で大文字小文字の扱いを明示しない。
- CSV のように引用符やカンマを含む可能性がある形式を、単純な `Split(',')` だけで処理しようとする。

## 練習問題

1. 名前を入力し、前後の空白を取り除いて表示する。
2. 空白だけの入力をエラーにする。
3. `start`、`STOP`、`List` のような command を大文字小文字を無視して判定する。
4. カンマ区切りの文字列を分割し、各項目を `Trim()` して表示する。
