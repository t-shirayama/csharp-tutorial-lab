# スコープと変数の寿命

## 目的

変数を参照できる範囲と、値をどこまで保持すべきかを判断できるようにします。

## 前提

- [変数と型](02_変数と型.md) を読んでいる
- [繰り返し](04_繰り返し.md) を読んでいる

## 要点

- `{ }` の内側で宣言した変数は、基本的にそのブロック内だけで使えます。
- 変数の範囲はできるだけ狭くします。
- 長く生きる変数ほど、状態の変化を追いにくくなります。

## コード例

```csharp
// この例では「スコープと変数の寿命」の要点を最小のコードで確認します。
if (DateTimeOffset.UtcNow.Hour < 12)
{
    var message = "Good morning";
    Console.WriteLine(message);
}

// Console.WriteLine(message); // ここでは使えない
```

## コードの読み方

このコード例は「スコープと変数の寿命」の基本形を確認するためのものです。上から順に、値や object を用意し、C# の構文や .NET API を使い、最後に結果を確認します。まず入力、処理、出力の 3 つに分けて読むと、初学者でも流れを追いやすくなります。

## 実務での使い方

変数を必要な直前で宣言すると、コードを読む人が「この値はどこで変わるのか」を追いやすくなります。メソッド全体で使う変数が多い場合、メソッド分割のサインです。

## よくあるミス

- メソッド先頭にすべての変数を宣言する。
- ループ内で使う一時変数を外側に出す。
- 同じ名前の変数を別スコープで使い、読み手を混乱させる。

## 練習問題

1. `if` ブロック内の変数を外から参照できないことを確認する。
2. ループ内だけで使う変数をループ内へ移す。
3. 長いメソッドから一時変数の寿命を短くする。
