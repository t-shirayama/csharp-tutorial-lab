# 条件分岐

## 目的

条件によって処理を分けるための `if`、`else`、`switch`、`switch expression` を理解します。

## 前提

- [変数と型](02_変数と型.md) を読んでいる
- `bool` の値が `true` または `false` であることを知っている

## 要点

- `if` は条件が真のときだけ処理を実行します。
- `else if` と `else` で複数の条件を扱えます。
- 値の種類で分岐する場合は `switch` が読みやすいことがあります。
- 値を返す分岐には `switch expression` が便利です。

## if の基本

```csharp
// この例では「条件分岐」の要点を最小のコードで確認します。
var score = 82;

if (score >= 80)
{
    Console.WriteLine("合格です。");
}
else
{
    Console.WriteLine("再提出です。");
}
```

## else if

```csharp
// この例では「条件分岐」の要点を最小のコードで確認します。
var score = 72;

if (score >= 90)
{
    Console.WriteLine("A");
}
else if (score >= 70)
{
    Console.WriteLine("B");
}
else if (score >= 60)
{
    Console.WriteLine("C");
}
else
{
    Console.WriteLine("D");
}
```

条件は上から順に評価されます。順番が変わると結果も変わることがあります。

## switch statement

```csharp
// この例では「条件分岐」の要点を最小のコードで確認します。
var command = "start";

switch (command)
{
    case "start":
        Console.WriteLine("開始します。");
        break;
    case "stop":
        Console.WriteLine("停止します。");
        break;
    default:
        Console.WriteLine("不明なコマンドです。");
        break;
}
```

## switch expression

```csharp
// この例では「条件分岐」の要点を最小のコードで確認します。
var command = "start";

var message = command switch
{
    "start" => "開始します。",
    "stop" => "停止します。",
    _ => "不明なコマンドです。"
};

Console.WriteLine(message);
```

値を作るための分岐なら、`switch expression` の方が簡潔に書けます。

## よくあるミス

- `=` と `==` を間違える。
- `else if` の順序を間違えて、先に広い条件が一致してしまう。
- `switch` の `default` や `switch expression` の `_` を忘れて、想定外の値を扱えない。
- 条件式を長くしすぎて、何を判定しているか分かりにくくなる。

## コードの読み方

このコード例は「条件分岐」の基本形を確認するためのものです。上から順に、値や object を用意し、C# の構文や .NET API を使い、最後に結果を確認します。まず入力、処理、出力の 3 つに分けて読むと、初学者でも流れを追いやすくなります。

## 実務での使い方

実務では、入力値の検証、権限の判定、状態ごとの処理、エラーの分類などで条件分岐を使います。条件が増えすぎたら、メソッド化、クラス化、辞書、Strategy パターンなどを検討します。

## 練習問題

1. 点数を入力して、A/B/C/D の評価を表示する。
2. コマンド文字列 `add`, `delete`, `list` によって表示メッセージを変える。
3. `if` で書いた分岐を `switch expression` に書き換える。
