# ステートメントと式

## 目的

C# の文を読むときに、ステートメントと式の違いを理解できるようにします。

## 前提

- [変数と型](02_変数と型.md) を読んでいる
- [条件分岐](03_条件分岐.md) を読んでいる

## 要点

- ステートメントは処理の単位です。
- 式は値を生み出す構文です。
- `if` や `while` はステートメント、`a + b` や `name.Length` は式です。

## コード例

```csharp
// この例では「ステートメントと式」の要点を最小のコードで確認します。
int price = 1200;
int quantity = 2;

// price * quantity は値を返す式です。
int total = price * quantity;

// if 全体は処理を表すステートメントです。
if (total >= 2000)
{
    Console.WriteLine("送料無料です。");
}
```

## コードの読み方

`price * quantity` は計算結果という値になります。一方、`if` は条件に応じて処理を実行する構文です。コードを読むときは「値を作っているのか」「処理の流れを作っているのか」を分けて見ると理解しやすくなります。

## 実務での使い方

条件式、switch 式、null 合体演算子、三項演算子を読むときに必要です。式を詰め込みすぎると読みにくくなるため、業務ルールが複雑な場合は変数名を付けて分けます。

## よくあるミス

- 式を長くしすぎて、何を計算しているか分からなくなる。
- 副作用のある処理を式の中に混ぜる。
- ステートメントと式の違いを意識せず、switch 式と switch ステートメントを混同する。

## 練習問題

1. 自分のコードから式を 3 つ探す。
2. ステートメントを 3 つ探す。
3. 長い式に一時変数名を付けて読みやすくする。
