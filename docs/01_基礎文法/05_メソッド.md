# メソッド

## 目的

処理に名前を付けて再利用するためのメソッドを理解します。

## 前提

- [変数と型](02_変数と型.md) を読んでいる
- [条件分岐](03_条件分岐.md) を読んでいる
- [繰り返し](04_繰り返し.md) を読んでいる

## 要点

- メソッドは処理をまとめ、名前を付ける仕組みです。呼び出し側が中身を全部読まなくても意図を理解できるように、`CalculateTotal`、`ValidateEmail`、`SaveOrderAsync` のように目的が伝わる名前を付けます。
- 引数で外から値を受け取り、戻り値で結果を返します。引数はメソッドが処理に必要とする入力、戻り値は呼び出し側へ返す成果物として考えると、責務が整理しやすくなります。
- 戻り値がない場合は `void` を使います。ただし、計算結果や判定結果を呼び出し側で使う可能性があるなら、画面表示やログ出力までメソッド内で済ませず、戻り値として返す設計を優先します。
- 実務では、メソッド名、引数、戻り値が仕様の読みやすさを大きく左右します。呼び出し行だけで「何を渡し、何が返るか」が分かるメソッドは、テストもしやすく変更にも強くなります。
- 1つのメソッドに入力チェック、計算、保存、通知などを詰め込むと、変更理由が増えて読みにくくなります。処理のまとまりが複数ある場合は、目的ごとに小さなメソッドへ分けます。
- 例外を投げるのか、`bool` や結果 object で失敗を返すのかは呼び出し側の扱いやすさで決めます。入力ミスのように想定内の失敗と、DB 障害のような想定外の失敗は分けて考えます。

## 基本形

```csharp
// この例では「メソッド」の要点を最小のコードで確認します。
static int Add(int left, int right)
{
    return left + right;
}

var result = Add(10, 20);
Console.WriteLine(result);
```

`left` と `right` が引数、`int` が戻り値の型です。`return` で結果を返します。

## void メソッド

```csharp
// この例では「メソッド」の要点を最小のコードで確認します。
static void PrintGreeting(string name)
{
    Console.WriteLine($"こんにちは、{name}さん。");
}

PrintGreeting("Sato");
```

画面表示やログ出力のように、戻り値が不要な処理では `void` を使います。

## メソッドに分ける例

```csharp
// この例では「メソッド」の要点を最小のコードで確認します。
static decimal CalculateTotal(decimal unitPrice, int quantity)
{
    return unitPrice * quantity;
}

var total = CalculateTotal(1200m, 3);
Console.WriteLine($"合計: {total}円");
```

計算処理をメソッドにすると、呼び出し側は「何をしているか」を読みやすくなります。

## ガード句

```csharp
// この例では「メソッド」の要点を最小のコードで確認します。
static decimal CalculateDiscountedPrice(decimal price, decimal discountRate)
{
    if (price < 0)
    {
        throw new ArgumentOutOfRangeException(nameof(price), "価格は0以上にしてください。");
    }

    if (discountRate < 0 || discountRate > 1)
    {
        throw new ArgumentOutOfRangeException(nameof(discountRate), "割引率は0から1の範囲にしてください。");
    }

    return price * (1 - discountRate);
}
```

不正な入力を先に弾く書き方をガード句と呼びます。正常系の処理を読みやすく保てます。

## よくあるミス

- メソッド名が曖昧で、何をする処理か分からない。
- 1つのメソッドに複数の責務を詰め込みすぎる。
- 引数が多すぎて、呼び出し側が読みにくい。
- 戻り値を使わず、副作用だけに頼りすぎる。

## コードの読み方

`CalculateTotal` は `unitPrice` と `quantity` を入力として受け取り、計算結果を戻り値で返します。呼び出し側は `var total = CalculateTotal(1200m, 3);` だけを読めば「合計を計算している」と分かります。`Console.WriteLine` を計算メソッドの中に入れないことで、同じ計算を画面表示、テスト、Web API など別の場所でも使いやすくしています。

## 実務での使い方

実務では、メソッドは単なる文法ではなく設計の最小単位です。テストしやすいメソッドは、入力と出力が明確で、外部状態に依存しすぎません。長い処理を分けるときは、行数ではなく「名前を付ける価値がある処理か」で判断します。

入力、検証、計算をメソッドに分ける流れは [小さなコンソールアプリ実践](18_小さなコンソールアプリ実践.md) でまとめて確認できます。
