# デプロイ前チェックリスト

## 目的

本番反映前に確認すべき build、test、設定、migration、監視の観点を整理します。

## 前提

- [GitHub Actions](05_GitHubActions.md) を読んでいる
- [release と versioning](07_releaseとversioning.md) を読んでいる

## 要点

- デプロイ前確認は属人化させず、チェックリスト化します。
- DB migration、secret、環境変数、rollback を事前に確認します。
- リリース後の監視項目も決めます。

## チェックリスト

- `dotnet build` が通る。
- `dotnet test` が通る。
- migration の適用手順と戻し方を確認した。
- 本番 secret と環境変数が設定済み。
- OpenAPI や README の変更が反映済み。
- ログ、メトリクス、アラートを確認できる。
- rollback 手順がある。

## 実務での使い方

小さな個人開発でも、リリース手順を文章化すると事故を減らせます。チームでは PR template や release issue にチェックリストを含めます。

## よくあるミス

- DB migration の影響を本番直前まで確認しない。
- secret 未設定で起動後に失敗する。
- 失敗時の rollback 手順がない。
- 監視せずリリースして完了にする。
