# 公式ドキュメントの読み方

## 目的

Microsoft Learn や公式リファレンスから、必要な情報を早く正確に読む方法を整理します。

## 前提

- [用語と一次情報リンク集](05_用語と一次情報リンク集.md) を読んでいる

## 要点

- 対象バージョンを確認します。
- API reference と tutorial の違いを意識します。
- Warning、Important、Note は実務上の落とし穴が多いです。
- サンプルコードの前提条件を確認します。

## 読む順序

1. ページタイトルと対象技術を確認する。
2. 対象バージョンと更新日を見る。
3. 前提条件とサポート範囲を読む。
4. サンプルを最小 project で試す。
5. 関連リンクから一次情報へ辿る。

## 実務での使い方

外部記事で解決策を見つけた場合も、最終的には公式ドキュメントで対象バージョンと推奨方針を確認します。古い .NET Framework 向けの記事を .NET 10 にそのまま適用しないよう注意します。

## よくあるミス

- 対象バージョンを見ない。
- tutorial の簡略化されたコードを本番設計として使う。
- 英語ページと日本語ページの更新差分を見落とす。
- Stack Overflow の古い回答だけで判断する。
