# 18章アウトライン対応表

## 目的

提示された C# 詳説系の 18 章アウトラインに対して、このナレッジベースで何が対応済みで、何を追加すべきかを整理します。

## 前提

- このナレッジベースは、C# / .NET を実務で使える状態へ近づけることを主目的にしています。
- 提示アウトラインは、C# 言語仕様、CLR、ライブラリ内部、メモリ効率などの深掘りを多く含みます。
- 対応状況は、記事が存在するかだけでなく、初学者が読んで実務判断に使える粒度かどうかで判断します。

## 要点

- 基礎文法、OOP、LINQ、非同期、テスト、Web API、EF Core、CI/CD はかなり対応済みです。
- struct、enum、delegate、event、IDisposable、Stream、Span、Threading の入口記事を追加済みです。
- Rx、アセンブリ、GC、メモリ効率、低レベル同期プリミティブも入口レベルの記事を追加済みです。
- 18 章をそのままカテゴリ化せず、既存カテゴリと [13_ランタイムと高度なCSharp](../13_ランタイムと高度なCSharp/README.md) に分散して整理しています。

## 対応状況の凡例

| 記号 | 意味 |
| --- | --- |
| 対応済み | 既存記事で学習できる。 |
| 部分対応 | 入口はあるが、章立てほど細かくない。 |
| 未対応 | 主要な記事がまだない。 |

## 追加優先度

| 優先度 | 追加候補 | 理由 |
| --- | --- | --- |
| 高 | struct、enum、delegate、event、IDisposable、Stream | 実務コードや既存 API の読解で頻出する。 |
| 高 | generic constraints、variance、exception filter、rethrow | ライブラリ利用、レビュー、障害調査で判断材料になる。 |
| 中 | GC、boxing、Span、Memory、SynchronizationContext | 性能問題や非同期事故の理解に効く。 |
| 中 | assembly、AssemblyLoadContext、attribute / reflection 詳細 | プラグイン、DI、serializer、テスト補助で出る。 |
| 低 | Rx、LINQ to XML、Tx | 特定領域では重要だが、Web API 実務の初期優先度は低い。 |

## 実務での使い方

この対応表は、学習計画の抜け漏れ確認に使います。最初から 18 章を順番に全部読むより、現場で必要な順に既存記事を読み、薄い章だけ追加学習する方が効率的です。

実務即戦力を優先する場合は、Web API、EF Core、テスト、CI/CD を先に固めます。C# 言語・CLR の深掘りは、コードレビューで説明できない挙動や性能問題に出会ったタイミングで厚くしていきます。

## よくあるミス

- 書籍風の章立てをそのままカテゴリにして、既存の実務ロードマップと重複させる。
- GC や Span などの高度な話題を、初学者に最初から必須項目として扱う。
- Rx や低レベル threading を、必要なユースケースなしに先に深掘りする。
- 既存記事に対応済みの内容を、別名で重複追加する。
