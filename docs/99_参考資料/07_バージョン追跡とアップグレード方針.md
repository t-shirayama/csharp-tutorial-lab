# バージョン追跡とアップグレード方針

## 目的

.NET、C#、NuGet package、SDK、CI image のバージョンを追跡し、安全に更新する方針を持ちます。

## 前提

- [C# バージョン差分](02_CSharpバージョン差分.md) を読んでいる
- [.NET 8 / .NET 10 差分](03_dotnet8-dotnet10差分.md) を読んでいる

## 要点

- LTS と STS のサポート期限を確認します。
- SDK、runtime、language version、NuGet package を分けて考えます。
- 更新は小さく分け、build/test/動作確認を通します。
- breaking change と obsolete warning を確認します。

## 確認コマンド

```powershell
dotnet --info
dotnet --list-sdks
dotnet list package --outdated
```

## 実務での使い方

アップグレード PR では、対象、理由、影響、戻し方、確認コマンドを明記します。大きなバージョン更新では、まず CI を通し、次に主要機能の手動確認を行います。

## よくあるミス

- SDK 更新と NuGet 更新を一度に大量に行う。
- warning を無視して次の LTS まで放置する。
- CI image の更新漏れでローカルと差が出る。
- breaking change の移行手順を読まない。
