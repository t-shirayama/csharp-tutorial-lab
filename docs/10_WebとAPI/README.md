## 到達目標

- Web API の入口である `Program.cs` を読める。
- ルーティング、DI、ミドルウェアの関係を説明できる。
- CRUD API を作成できる。
- middleware、filter、versioning、外部 API 連携を設計できる。
- validation、認証方式、CORS、rate limiting、cache の実務上の注意を説明できる。
- HTTP status code、ProblemDetails、OpenAPI metadata を API 契約として整えられる。
- 一覧 API の paging / filtering / sorting と Web API 統合テストの入口を説明できる。
