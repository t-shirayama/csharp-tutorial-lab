# プロジェクト作成と実行

## 目的

`dotnet new` でプロジェクトを作り、restore、build、run、test の流れを説明できるようにします。

## 前提

- [.NET SDK](01_dotnet-sdk.md) がインストール済み
- PowerShell で作業できる

## 要点

- `dotnet new` はテンプレートから project を作るコマンドです。
- `dotnet restore` は NuGet パッケージを復元します。
- `dotnet build` はコンパイルしてエラーを確認します。
- `dotnet run` はアプリを実行します。
- 実務では solution root で作業できる構成にします。

## コマンド例

```powershell
dotnet new console -n HelloCsharp
Set-Location HelloCsharp
dotnet restore
dotnet build
dotnet run
```

## solution を使う例

```powershell
dotnet new sln -n SampleApp
dotnet new console -n SampleApp.Console -o src/SampleApp.Console
dotnet sln add src/SampleApp.Console/SampleApp.Console.csproj
dotnet build
```

test project も含めた実務寄りの構成は、[テストプロジェクト付き solution](09_テストプロジェクト付きsolution.md) で扱います。

## 実務での使い方

小さな検証は単体 project で始めてもよいですが、アプリ本体、テスト、共通ライブラリが増える前提なら solution を先に作ります。README には「どのディレクトリで、どのコマンドを実行するか」を残します。

## よくあるミス

- solution root ではなく project directory でしかコマンドが通らない構成にする。
- `bin` や `obj` を手で編集する。
- エラーを読まずに `dotnet clean` だけ繰り返す。
- template 名と project 名を混同する。

## 練習問題

1. console project を作成して `dotnet run` する。
2. solution を作成して project を追加する。
3. solution root で `dotnet build` が通ることを確認する。
4. 失敗したコマンドとエラーメッセージをメモする。
