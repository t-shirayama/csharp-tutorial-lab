# インストール手順チェックリスト

## 目的

C# / .NET 学習を始めるために、何をどこから入れて、どの設定にするかを順番に確認します。

## 前提

- Windows 10 または Windows 11
- PowerShell を起動できる
- インターネットに接続できる

## 全体の順序

1. .NET SDK を入れる。
2. Visual Studio Code を入れる。
3. C# Dev Kit を入れる。
4. PowerShell と VS Code を開き直す。
5. `dotnet` と VS Code の動作確認をする。

## 1. .NET SDK を入れる

### .NET SDK を公式サイトから入れる場合

次の公式ページを開きます。

- [.NET 10.0 downloads](https://dotnet.microsoft.com/download/dotnet/10.0)

選ぶもの:

| 項目 | 選択 |
| --- | --- |
| 種類 | SDK |
| OS | Windows |
| CPU | x64 |
| installer | Windows x64 installer |

Runtime ではなく **SDK** を選びます。

SDK には開発に必要な runtime も含まれるため、学習・開発用 PC では SDK を入れれば十分です。システム全体へインストールする場合は、UAC や管理者権限の確認が出ることがあります。

### .NET SDK をコマンドで入れる場合

PowerShell を開き、次を実行します。

```powershell
winget install --id Microsoft.DotNet.SDK.10 --source winget
```

現場や教材が .NET 8 LTS を指定している場合は、次も入れます。

```powershell
winget install --id Microsoft.DotNet.SDK.8 --source winget
```

### インストール時の設定

| 画面 | 設定 |
| --- | --- |
| インストール先 | 既定のまま |
| PATH | installer に任せる。手動で編集しない |
| 追加 workload | 最初は不要 |
| 完了後 | PowerShell と VS Code を開き直す |

## 2. Visual Studio Code を入れる

### VS Code を公式サイトから入れる場合

次の公式ページを開きます。

- [Visual Studio Code](https://code.visualstudio.com/)

選ぶもの:

| 項目 | 選択 |
| --- | --- |
| installer | Windows User Installer x64 |
| インストール先 | 既定のまま |
| PATH 追加 | 有効 |
| `Open with Code` | 有効推奨 |

### VS Code をコマンドで入れる場合

```powershell
winget install --id Microsoft.VisualStudioCode --source winget
```

## 3. C# Dev Kit を入れる

VS Code を開き、Extensions で **C# Dev Kit** を検索してインストールします。

C# Dev Kit は C# 拡張と .NET Install Tool も自動で入れます。現在の要件は .NET 10 SDK なので、先に .NET SDK の確認を済ませておきます。

コマンドで入れる場合は次を実行します。

```powershell
code --install-extension ms-dotnettools.csdevkit
```

`code` が見つからない場合は、VS Code を再起動してからもう一度試します。

## 4. 動作確認

PowerShell を新しく開き、次を実行します。

```powershell
dotnet --version
dotnet --list-sdks
dotnet --info
code --version
```

期待する状態:

- `dotnet --version` で `10.` から始まる version が表示される。
- `dotnet --list-sdks` に `10.0.xxx` が表示される。
- `code --version` で VS Code の version が表示される。

## 5. 最小アプリで確認する

```powershell
dotnet new console -n EnvCheck
Set-Location EnvCheck
dotnet run
code .
```

期待する状態:

- `dotnet run` で `Hello, World!` が表示される。
- VS Code で `Program.cs` が開ける。
- C# の色分け、補完、エラー表示が動く。

## よくあるミス

- Runtime だけを入れて SDK を入れていない。
- x86 installer を選んでしまう。通常は x64 を選ぶ。
- インストール後に古い PowerShell や VS Code を使い続ける。
- VS Code で project folder ではなく単体 `.cs` ファイルだけ開く。
- C# Dev Kit を入れずに補完やデバッグが効かないと思い込む。

## 完了条件

- `dotnet --info` が表示できる。
- `dotnet new console` が成功する。
- `dotnet run` が成功する。
- VS Code で C# Dev Kit が有効になっている。
