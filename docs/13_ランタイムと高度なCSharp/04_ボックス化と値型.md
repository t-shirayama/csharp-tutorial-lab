# ボックス化と値型

## 目的

値型が `object` や interface として扱われるときに起きる boxing を理解します。

## 前提

- [struct](../02_オブジェクト指向/11_struct.md) を読んでいる

## 要点

- boxing は値型を object として heap 上に包む処理です。
- unboxing は object から元の値型へ取り出す処理です。
- 頻繁な boxing は allocation と性能問題につながります。

## コード例

```csharp
// この例では「ボックス化と値型」の要点を最小のコードで確認します。
int number = 42;

// int が object として扱われるため boxing が発生します。
object boxed = number;

// 取り出すときは unboxing が発生します。
int unboxed = (int)boxed;

Console.WriteLine(unboxed);
```

## コードの読み方

`int` は値型ですが、`object` に代入すると object として扱うために boxing されます。generic collection を使わず `ArrayList` のような非 generic collection を使うと boxing が増えます。

## 実務での使い方

通常の業務コードでは過度に気にしすぎる必要はありません。ただし high throughput な logging、formatting、数値処理、古い非 generic API を扱うときは allocation を確認します。

## よくあるミス

- boxing が起きる場所を知らず、hot path で allocation を増やす。
- unboxing で違う型に cast して例外にする。
- nullable value type の boxing 仕様を誤解する。

## 練習問題

1. `int` を `object` に代入する。
2. `List<int>` と `ArrayList` の違いを調べる。
3. interface 経由で struct を扱う場合の boxing 可能性を調べる。
