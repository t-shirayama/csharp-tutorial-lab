# 完成度チェックリスト

## 目的

「動いた」で終わらせず、実務で扱える品質に近づける観点を確認します。

## 作るもの

実装済みアプリに対する品質チェックリストです。

## 学習ポイント

- 正常系だけでなく異常系を見る
- テスト、ログ、設定、エラー、セキュリティを確認する
- README と運用手順を整える

## チェックリスト

- 入力 validation がある。
- エラー時の response や message が分かりやすい。
- 重要な処理に structured log がある。
- secret が repository とログに出ていない。
- 設定値が環境ごとに切り替えられる。
- unit test と必要な integration test がある。
- `dotnet format` または同等の整形を実行した。
- README に起動、テスト、設定、トラブルシュートがある。

## 完了条件

- 初見の人が README だけで起動できる。
- 失敗時にログとエラーメッセージから調査できる。
- 変更時にテストで主要仕様を守れる。
