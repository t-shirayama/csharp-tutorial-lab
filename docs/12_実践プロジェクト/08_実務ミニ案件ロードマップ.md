# 実務ミニ案件ロードマップ

## 目的

学習用アプリを、実務の小さな案件として進めるための順序を整理します。

## 作るもの

TODO、家計簿、メモ API などを題材に、要件整理から PR 提出までを一通り行います。

## 学習ポイント

- 要件を小さな issue に分ける
- API、DB、テスト、ログ、設定を段階的に追加する
- README に実行手順と確認観点を残す
- PR とレビューを想定して変更を説明する

## 手順

1. 作る機能を3つ以内に絞る。
2. 入力、出力、エラー、保存先を決める。
3. 最小実装を作る。
4. テストを追加する。
5. ログ、設定、例外処理を整える。
6. README と確認コマンドを更新する。
7. PR 提出チェックを行う。

## 完了条件

- `dotnet build` と `dotnet test` が通る。
- README の手順で動作確認できる。
- 主要な正常系、異常系、境界値のテストがある。
- ログとエラーレスポンスが調査に使える。
