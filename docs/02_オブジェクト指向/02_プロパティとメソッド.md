# プロパティとメソッド

## 目的

クラスが持つ値を表すプロパティと、振る舞いを表すメソッドを使い分けられるようにします。

## 前提

- [class と object](01_classとobject.md) を読んでいる
- [メソッド](../01_基礎文法/05_メソッド.md) を読んでいる

## 要点

- プロパティは、オブジェクトが持つ状態や値を表します。
- メソッドは、オブジェクトが行う処理や判断を表します。
- `get` は読み取り、`set` は書き込みを表します。
- 実務では、外から自由に変更されてよい値かどうかを意識します。

## プロパティ

```csharp
// この例では「プロパティとメソッド」の要点を最小のコードで確認します。
var product = new Product
{
    Name = "Mouse",
    Price = 3000m
};

Console.WriteLine(product.Name);
Console.WriteLine(product.Price);

public class Product
{
    public string Name { get; set; } = "";
    public decimal Price { get; set; }
}
```

`get; set;` があるプロパティは、外から読み書きできます。

## メソッド

```csharp
// この例では「プロパティとメソッド」の要点を最小のコードで確認します。
var product = new Product
{
    Name = "Mouse",
    Price = 3000m
};

var priceWithTax = product.CalculatePriceWithTax(0.1m);
Console.WriteLine(priceWithTax);

public class Product
{
    public string Name { get; set; } = "";
    public decimal Price { get; set; }

    public decimal CalculatePriceWithTax(decimal taxRate)
    {
        return Price * (1 + taxRate);
    }
}
```

`CalculatePriceWithTax` は、`Product` の価格を使って税込価格を計算する振る舞いです。

## 読み取り専用プロパティ

外から変更されたくない値は、`private set` や getter only のプロパティにします。

```csharp
// この例では「プロパティとメソッド」の要点を最小のコードで確認します。
var order = new Order();
order.AddItem(1200m);
order.AddItem(800m);

Console.WriteLine(order.TotalAmount);

public class Order
{
    public decimal TotalAmount { get; private set; }

    public void AddItem(decimal price)
    {
        TotalAmount += price;
    }
}
```

`TotalAmount` は外から読めますが、変更できるのは `Order` クラスの中だけです。

## 計算プロパティ

値を保存せず、他の値から計算して返すプロパティも書けます。

```csharp
// この例では「プロパティとメソッド」の要点を最小のコードで確認します。
public class Person
{
    public string FirstName { get; set; } = "";
    public string LastName { get; set; } = "";

    public string FullName => $"{LastName} {FirstName}";
}
```

`FullName` は保存された値ではなく、`LastName` と `FirstName` から毎回作られます。

## よくあるミス

- すべてのプロパティを無条件に `set` 可能にする。
- メソッド名が曖昧で、何をするのか分からない。
- 計算で求められる値を別プロパティに保存し、値の不整合を作る。
- クラス外でやるべき処理と、クラス内に持たせるべき処理の境界を考えない。

## コードの読み方

このコード例は「プロパティとメソッド」の基本形を確認するためのものです。上から順に、値や object を用意し、C# の構文や .NET API を使い、最後に結果を確認します。まず入力、処理、出力の 3 つに分けて読むと、初学者でも流れを追いやすくなります。

## 実務での使い方

業務ロジックでは、状態を守ることが重要です。注文金額や在庫数のように勝手に書き換わると困る値は、外から直接 `set` できない形にして、意味のあるメソッド経由で変更します。

## 練習問題

1. `Product` クラスに商品名、価格、税込価格を返すメソッドを作る。
2. `Order` クラスに明細追加メソッドと合計金額プロパティを作る。
3. `Person` クラスに氏名を結合する計算プロパティを作る。
4. 外から変更できるべき値と、変更できない方がよい値を分類する。
