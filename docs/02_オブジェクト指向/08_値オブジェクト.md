# 値オブジェクト

## 目的

メールアドレス、金額、期間など、値そのものに意味がある概念を型として表現できるようにします。

## 前提

- [record](04_record.md) を読んでいる
- [カプセル化](07_カプセル化.md) を読んでいる

## 要点

- 値オブジェクトは値が同じなら同じものとして扱います。
- 不正な値を作れないように constructor や factory で検証します。
- `record` は値の等価性を表現しやすい機能です。

## コード例

```csharp
// この例では「値オブジェクト」の要点を最小のコードで確認します。
public sealed record EmailAddress
{
    public string Value { get; }

    public EmailAddress(string value)
    {
        if (string.IsNullOrWhiteSpace(value) || !value.Contains('@'))
        {
            throw new ArgumentException("Invalid email address.", nameof(value));
        }

        Value = value;
    }

    public override string ToString() => Value;
}
```

## コードの読み方

このコード例は「値オブジェクト」の基本形を確認するためのものです。上から順に、値や object を用意し、C# の構文や .NET API を使い、最後に結果を確認します。まず入力、処理、出力の 3 つに分けて読むと、初学者でも流れを追いやすくなります。

## 実務での使い方

文字列や数値をそのまま渡すと、単位、形式、意味を間違えやすくなります。値オブジェクトにすると、メソッドの引数だけで意図が伝わりやすくなります。

## よくあるミス

- すべての値を値オブジェクトにして過剰設計になる。
- validation を外側に散らばらせる。
- Entity と Value Object の違いを混同する。

## 練習問題

1. `Money` 型を作り、金額が負にならないようにする。
2. `EmailAddress` 型を使う service を作る。
3. 文字列引数と値オブジェクト引数の読みやすさを比較する。
