# フィールドとインデクサ

## 目的

field と property の違い、indexer の基本を理解します。

## 前提

- [プロパティとメソッド](02_プロパティとメソッド.md) を読んでいる

## 要点

- field は class や struct が持つ変数です。
- public field より property を優先します。
- indexer は `obj[index]` の形で値へアクセスする仕組みです。

## コード例

```csharp
// この例では「フィールドとインデクサ」の要点を最小のコードで確認します。
public class OrderLines
{
    private readonly List<string> lines = [];

    public string this[int index] => lines[index];

    public int Count => lines.Count;

    public void Add(string line)
    {
        lines.Add(line);
    }
}

var orderLines = new OrderLines();
orderLines.Add("Keyboard");
Console.WriteLine(orderLines[0]);
```

## コードの読み方

`lines` は内部状態を持つ private field です。外部には `Count` property と indexer だけを公開しています。これにより、内部の `List<string>` を直接変更されることを防ぎます。

## 実務での使い方

field は基本的に private にし、外部に見せる値は property にします。indexer は collection に近い独自型で便利ですが、何を index にしているか分かりにくい場合は method 名を付けた方が読みやすくなります。

## よくあるミス

- public field で内部状態を直接変更できるようにする。
- indexer に重い処理や外部 I/O を入れる。
- 範囲外 index の扱いを考えない。
