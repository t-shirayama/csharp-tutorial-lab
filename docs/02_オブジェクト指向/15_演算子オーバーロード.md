# 演算子オーバーロード

## 目的

独自型で `+` や `==` などの演算子を定義する方法と、使いどころを理解します。

## 前提

- [struct](11_struct.md) を読んでいる
- [値オブジェクト](08_値オブジェクト.md) を読んでいる

## 要点

- 演算子オーバーロードは、独自型に自然な演算を定義する仕組みです。
- 数値、金額、ベクトル、範囲などで向いています。
- 意味が直感的でない演算子は避けます。

## コード例

```csharp
// この例では「演算子オーバーロード」の要点を最小のコードで確認します。
public readonly record struct Quantity(int Value)
{
    public static Quantity operator +(Quantity left, Quantity right)
    {
        return new Quantity(left.Value + right.Value);
    }
}

var first = new Quantity(2);
var second = new Quantity(3);
var total = first + second;

Console.WriteLine(total.Value);
```

## コードの読み方

`operator +` により、`Quantity` 同士を `+` で足せます。演算子の意味が型の意味と一致しているため読みやすくなります。一方、業務処理の実行や保存のような副作用を演算子に入れるのは避けます。

## 実務での使い方

値オブジェクトや数値系の型で使います。`==` を定義する場合は `Equals`、`GetHashCode`、`IEquatable<T>` との整合性も考えます。record struct は値の等価性を扱いやすいため、単純な値型では候補になります。

## よくあるミス

- 演算子に直感と違う意味を持たせる。
- `==` と `Equals` の結果が矛盾する。
- null や overflow の扱いを考えない。
