# コンストラクタ

## 目的

オブジェクトを作るときに必要な値を受け取り、不完全な状態のオブジェクトを作りにくくする方法を理解します。

## 前提

- [class と object](01_classとobject.md) を読んでいる
- [プロパティとメソッド](02_プロパティとメソッド.md) を読んでいる

## 要点

- コンストラクタは、オブジェクトを作るときに呼ばれる特別なメソッドです。
- クラス名と同じ名前で、戻り値の型を書きません。
- 必須の値はコンストラクタで受け取ると、初期化忘れを減らせます。
- 実務では、不正な状態を作れない設計に近づけるために使います。

## 基本例

```csharp
// この例では「コンストラクタ」の要点を最小のコードで確認します。
var product = new Product("Keyboard", 12000m);

Console.WriteLine(product.Name);
Console.WriteLine(product.Price);

public class Product
{
    public Product(string name, decimal price)
    {
        Name = name;
        Price = price;
    }

    public string Name { get; }
    public decimal Price { get; }
}
```

`Name` と `Price` はコンストラクタで必ず設定されます。

## 入力検証

```csharp
// この例では「コンストラクタ」の要点を最小のコードで確認します。
var product = new Product("Keyboard", 12000m);

public class Product
{
    public Product(string name, decimal price)
    {
        if (string.IsNullOrWhiteSpace(name))
        {
            throw new ArgumentException("商品名は必須です。", nameof(name));
        }

        if (price < 0)
        {
            throw new ArgumentOutOfRangeException(nameof(price), "価格は0以上にしてください。");
        }

        Name = name;
        Price = price;
    }

    public string Name { get; }
    public decimal Price { get; }
}
```

オブジェクトを作る時点で検証すると、不正な商品が後続処理へ流れにくくなります。

## デフォルトコンストラクタ

コンストラクタを1つも書かない場合、引数なしで作れるデフォルトコンストラクタが使えます。

```csharp
// この例では「コンストラクタ」の要点を最小のコードで確認します。
var user = new User();

public class User
{
    public string Name { get; set; } = "";
}
```

自分でコンストラクタを書くと、必要に応じて引数なしのコンストラクタも明示的に書く必要があります。

## primary constructor の入口

新しい C# では、クラス定義のところにコンストラクタ引数を書く primary constructor も使えます。

```csharp
// この例では「コンストラクタ」の要点を最小のコードで確認します。
public class Product(string name, decimal price)
{
    public string Name { get; } = name;
    public decimal Price { get; } = price;
}
```

短く書けますが、チームやプロジェクトの C# バージョン、可読性の方針に合わせて使います。

## よくあるミス

- 必須プロパティを `set` 可能にしたままにして、初期化忘れを許してしまう。
- コンストラクタで重い I/O 処理や外部 API 呼び出しを行う。
- 引数が多すぎて、何を渡しているか分かりにくくなる。
- 検証せずに不正な値をそのまま受け入れる。

## コードの読み方

このコード例は「コンストラクタ」の基本形を確認するためのものです。上から順に、値や object を用意し、C# の構文や .NET API を使い、最後に結果を確認します。まず入力、処理、出力の 3 つに分けて読むと、初学者でも流れを追いやすくなります。

## 実務での使い方

実務では、ドメイン上あり得ない状態を作れないようにすることが重要です。たとえば「名前のない商品」「負の価格」「明細のない確定注文」などを作れてしまうと、後の処理でバグが広がります。コンストラクタは、その入口で状態を守るための基本手段です。
