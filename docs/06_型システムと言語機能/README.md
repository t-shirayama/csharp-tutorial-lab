## 到達目標

- null 安全性を意識してコードを書ける。
- generic な型やメソッドを読める。
- pattern matching を使って条件分岐を簡潔に書ける。
- `required`、`init`、record、primary constructor を設計意図に合わせて選べる。
- generic constraints、variance、attribute、reflection の実務上の注意点を説明できる。
- `ref` / `out` / `in` と `Nullable<T>` を既存 API の読解で説明できる。
- `class`、`record`、`record struct`、`struct`、`enum` を用途に応じて選べる。
- `Result<T>` や attributes / reflection を実務で使う場面と避ける場面を説明できる。
