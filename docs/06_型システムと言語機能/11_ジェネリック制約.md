# ジェネリック制約

## 目的

generic type parameter に条件を付ける generic constraints を理解します。

## 前提

- [generics](02_generics.md) を読んでいる

## 要点

- `where T : class` は参照型に制限します。
- `where T : struct` は値型に制限します。
- `where T : new()` は public parameterless constructor を要求します。
- 制約により、generic code 内で使える member や前提が増えます。

## コード例

```csharp
// この例では「ジェネリック制約」の要点を最小のコードで確認します。
public static T Create<T>() where T : new()
{
    return new T();
}

var list = Create<List<string>>();
Console.WriteLine(list.Count);
```

## interface 制約の例

```csharp
// この例では「ジェネリック制約」の要点を最小のコードで確認します。
public static string Format<T>(T value) where T : IFormattable
{
    return value.ToString(null, null);
}
```

## コードの読み方

`where T : new()` があるため、method 内で `new T()` が使えます。`IFormattable` 制約があると、`T` を `IFormattable` として扱えます。

## 実務での使い方

repository、factory、serializer、validation helper、汎用 component で使います。制約が強すぎると使える型が減るため、本当に必要な前提だけを書きます。

## よくあるミス

- 制約なしで `T` の member を呼ぼうとする。
- `new()` 制約で DI が必要な型まで作ろうとする。
- `class`、`notnull`、nullable reference types の関係を混同する。

## 練習問題

1. `where T : class` の method を作る。
2. `where T : IComparable<T>` で比較できる method を作る。
3. 制約を付けすぎた場合の使いにくさを考える。
