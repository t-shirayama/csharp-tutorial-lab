# 属性の詳細

## 目的

attribute target、自作 attribute、reflection での取得方法を理解します。

## 前提

- [attributes](05_attributes.md) を読んでいる
- [reflection](06_reflection.md) を読んでいる

## 要点

- attribute は型、method、property、assembly などに付けられます。
- 自作 attribute は `Attribute` を継承します。
- attribute を付けるだけでは動かず、読む側の処理が必要です。

## コード例

```csharp
// この例では「属性の詳細」の要点を最小のコードで確認します。
[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method)]
public sealed class AuditAttribute : Attribute
{
    public AuditAttribute(string action)
    {
        Action = action;
    }

    public string Action { get; }
}

[Audit("注文を作成する")]
public class CreateOrderHandler
{
}
```

## 取得例

```csharp
// この例では「属性の詳細」の要点を最小のコードで確認します。
var attribute = typeof(CreateOrderHandler)
    .GetCustomAttributes(typeof(AuditAttribute), inherit: false)
    .OfType<AuditAttribute>()
    .FirstOrDefault();

Console.WriteLine(attribute?.Action);
```

## コードの読み方

`AuditAttribute` は class と method にだけ付けられるようにしています。`GetCustomAttributes` で reflection 経由に取得できます。framework の多くは、このように attribute を読み取って動作を変えます。

## 実務での使い方

xUnit、ASP.NET Core、serializer、validation、source generator、analyzer でよく見かけます。自作 attribute は便利ですが、処理の流れが見えにくくなるため、濫用しないようにします。

## よくあるミス

- attribute を付ければ自動で動くと思う。
- `AttributeUsage` を指定せず、意図しない場所に付けられる。
- reflection を大量に使い、性能やリファクタリング耐性を落とす。

## 練習問題

1. method に付ける `RequiresRoleAttribute` を作る。
2. reflection でその attribute を取得する。
3. attribute ではなく DI や設定で表すべき例を考える。
