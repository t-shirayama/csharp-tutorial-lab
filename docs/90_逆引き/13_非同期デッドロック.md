# 非同期デッドロック

## 症状

`.Result` や `.Wait()` を使った箇所で処理が止まる、UI が固まる、または request が返らなくなります。

## 主な原因

- 非同期処理を同期的に待っている。
- `SynchronizationContext` がある環境で continuation が戻れなくなっている。
- thread pool を block し、後続処理が実行されない。

## 確認コマンド

```powershell
dotnet build
dotnet test
```

## 対処例

```csharp
// この例では同期的に待たず、呼び出し元まで async を伝播させます。
public async Task<Product> GetProductAsync(int id, CancellationToken cancellationToken)
{
    var response = await httpClient.GetAsync($"products/{id}", cancellationToken);
    response.EnsureSuccessStatusCode();

    return await response.Content.ReadFromJsonAsync<Product>(cancellationToken)
        ?? throw new InvalidOperationException("Response body was empty.");
}
```

## コードの読み方

`GetAsync` と `ReadFromJsonAsync` の両方を `await` し、同期 block を作っていません。呼び出し元も `await GetProductAsync(...)` にすることで、非同期の流れを保ちます。

## 解決手順

1. `.Result`、`.Wait()`、`GetAwaiter().GetResult()` を検索する。
2. 呼び出し元を `async Task` に変更できるか確認する。
3. UI / legacy ASP.NET / test code で同期 block していないか確認する。
4. timeout と cancellation token を追加し、永久待ちを避ける。
5. thread pool exhaustion の可能性がある場合はログで同時実行数を確認する。
