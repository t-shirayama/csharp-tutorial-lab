# ビルドエラーの読み方

## 症状

`dotnet build` でエラーが出て、どこから直すべきか分からない状態です。

## 確認コマンド

```powershell
dotnet build
```

## 読む順番

1. 最初のエラーを見る。
2. ファイル名と行番号を確認する。
3. エラーコードを確認する。
4. 後続エラーが連鎖していないか見る。

## よくある原因

- セミコロン忘れ。
- 型名や変数名の typo。
- using 不足。
- nullable warning を error として扱っている。
