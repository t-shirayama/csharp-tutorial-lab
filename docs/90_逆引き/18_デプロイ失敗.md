# デプロイ失敗

## 症状

GitHub Actions やデプロイ環境で、build、test、image 作成、起動、health check のいずれかが失敗する。

## 主な原因

- CI と local の .NET SDK version が違う。
- Release build だけで warning や trimming の問題が出る。
- secret や environment variable が設定されていない。
- Docker image の tag、port、entrypoint がずれている。
- DB 接続、migration、権限、network が本番環境でだけ失敗している。

## 確認コマンド

```powershell
dotnet restore
dotnet build --configuration Release --no-restore
dotnet test --configuration Release --no-build
docker build -t app:local .
docker run --rm -p 8080:8080 app:local
```

## 切り分け

| 失敗地点 | 見るもの |
| --- | --- |
| restore | NuGet source、認証、package version |
| build | SDK version、warning、生成物 |
| test | 失敗 test、環境依存、外部依存 |
| image build | Dockerfile、copy path、publish output |
| app start | appsettings、secret、port、DB 接続 |
| health check | routing、network、dependency、timeout |

## 解決手順

1. 失敗した step 名と command を確認する。
2. local で同じ configuration の command を実行する。
3. 必須設定と secret が入っているか確認する。
4. container log とアプリ log を確認する。
5. 復旧優先なら rollback、原因調査なら staging で再現する。

## よくあるミス

- Debug だけ確認して Release を確認しない。
- CI log の最初の失敗ではなく、最後の例外だけを見る。
- secret の値を log に出して確認しようとする。
- rollback せずに本番へ修正 commit を重ねる。
