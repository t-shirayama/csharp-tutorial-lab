# DB 製品差分

## 目的

SQL Server、PostgreSQL、SQLite などの DB 製品差分を理解し、開発環境と本番環境の違いで起きる不具合を避けます。

## 前提

- [EF Core の基本](03_EFCoreの基本.md) を読んでいる
- [Migration](05_Migration.md) を読んでいる
- [接続文字列と Secret](10_接続文字列とSecret.md) を読んでいる

## 要点

- EF Core を使っても、DB 製品差分は消えません。
- connection string、型 mapping、文字列比較、大文字小文字、日付時刻、transaction、migration SQL が変わります。
- SQLite は軽量ですが、本番 DB の完全な代替ではありません。
- PostgreSQL と SQL Server では SQL 方言、index、identity、JSON、日時型に違いがあります。
- 本番と違う DB で test する場合は、差分を明示しておきます。

## 接続文字列の例

```json
{
  "ConnectionStrings": {
    "SqlServer": "Server=localhost;Database=AppDb;Trusted_Connection=True;TrustServerCertificate=True",
    "Postgres": "Host=localhost;Database=appdb;Username=app;Password=change-me",
    "Sqlite": "Data Source=app.db"
  }
}
```

## provider を切り替える例

```csharp
var provider = builder.Configuration["Database:Provider"];
var connectionString = builder.Configuration.GetConnectionString(provider ?? "Sqlite");

builder.Services.AddDbContext<AppDbContext>(options =>
{
    if (provider == "SqlServer")
    {
        options.UseSqlServer(connectionString);
    }
    else if (provider == "Postgres")
    {
        options.UseNpgsql(connectionString);
    }
    else
    {
        options.UseSqlite(connectionString);
    }
});
```

## 差分の見方

| 観点 | SQL Server | PostgreSQL | SQLite |
| --- | --- | --- | --- |
| 開発用途 | Windows 開発と業務システムで多い | Linux / container と相性が良い | local test や小規模用途に向く |
| 大文字小文字 | collation の影響を受ける | identifier は小文字化されやすい | 型が緩く差分が隠れやすい |
| JSON | JSON 関数を利用 | `jsonb` が強い | JSON 関数はあるが本番差分に注意 |
| migration | provider 固有 SQL | provider 固有 SQL | 制約変更に制限が出やすい |

## コードの読み方

`Database:Provider` で provider を選び、`UseSqlServer`、`UseNpgsql`、`UseSqlite` を切り替えています。実務では provider ごとの package が必要です。connection string は secret を含むため、repository に直接書かず、環境変数や secret store から渡します。

## 実務での使い方

local では SQLite、本番では PostgreSQL のような構成は便利ですが、文字列比較、日時、migration、transaction の差分で問題が出ることがあります。重要な query、migration、transaction は本番に近い DB で integration test を行います。

## よくあるミス

- SQLite で通ったから本番 DB でも安全だと判断する。
- provider 固有 package を入れ忘れる。
- migration を複数 provider で共有できる前提にする。
- DB の timezone と application の `DateTimeOffset` 方針を決めない。
- collation の違いで検索結果の大小文字や並び順が変わる。
