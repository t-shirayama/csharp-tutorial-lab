# N+1 問題

## 目的

関連データ取得でクエリが大量発行される N+1 問題を理解します。

## 要点

- 1件取得後、関連データ取得が N 回発生する問題です。
- EF Core では `Include`、projection、明示的な集計で回避します。
- SQL ログを見てクエリ数を確認します。

## 悪い例のイメージ

```csharp
// この例では「N+1 問題」の要点を最小のコードで確認します。
foreach (var order in orders)
{
    Console.WriteLine(order.Items.Count);
}
```

遅延読み込みが有効な場合、注文ごとに明細取得が走る可能性があります。

## コードの読み方

このコード例は「N+1 問題」の基本形を確認するためのものです。上から順に、値や object を用意し、C# の構文や .NET API を使い、最後に結果を確認します。まず入力、処理、出力の 3 つに分けて読むと、初学者でも流れを追いやすくなります。

## 実務での使い方

一覧画面、API のリストレスポンス、帳票で起きやすいです。必要な形に `Select` で projection して取得すると、過剰な Entity 読み込みも減らせます。

## よくあるミス

- 開発データが少なく性能問題に気づかない。
- `Include` を増やしすぎて巨大クエリにする。
- SQL ログを確認しない。
