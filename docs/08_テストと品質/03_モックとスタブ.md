# モックとスタブ

## 目的

外部依存を差し替えて、テスト対象だけを確認する考え方を理解します。

## 要点

- スタブは決まった値を返す代用品です。
- モックは呼び出しを検証する代用品です。
- 何でもモックにするとテストが実装詳細に寄りすぎます。

## コード例

```csharp
// この例では「モックとスタブ」の要点を最小のコードで確認します。
public class FixedClock : IClock
{
    public DateTimeOffset Now { get; init; } = DateTimeOffset.Parse("2026-05-22T00:00:00Z");
}
```

## コードの読み方

`FixedClock` は現在時刻を固定する stub です。production code では現在時刻が変わりますが、テストでは `Now` を固定することで、実行時刻に左右されない検証ができます。値を返すだけなら、このような手書き stub の方が mock library より読みやすい場合があります。

## 実務での使い方

現在時刻、メール送信、外部 API、DB などを差し替えます。まず interface と DI で差し替えられる設計にしておくことが前提です。

Moq や NSubstitute のようなライブラリを使った実装例は [09_MoqとNSubstitute.md](09_MoqとNSubstitute.md) で扱います。

時刻や外部依存が原因でテストが不安定になる場合は [flaky test 対策](17_flaky test対策.md) も確認します。

## よくあるミス

- private メソッドまで無理にモックしようとする。
- 実装の呼び出し順に依存した壊れやすいテストを書く。
- モックが多すぎてテストの意図が読めない。

## レビュー観点

- モックしている依存は本当に外部依存か。
- 戻り値を固定するだけならスタブで十分ではないか。
- 呼び出し回数の検証が実装詳細に寄りすぎていないか。
- モックが多すぎて、テスト対象の責務が大きくなっていないか。
