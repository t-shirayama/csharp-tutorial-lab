## 到達目標

- 小さな単位でテストを書ける。
- 仕様をテスト名とアサーションで表現できる。
- 品質チェックを CLI で実行できる。
- テストプロジェクト、モック、カバレッジ、CI を実務の流れに組み込める。
- Unit / Integration / E2E の粒度を判断できる。
- 境界値、例外、非同期、キャンセルのテストを書ける。
- テストデータ管理、flaky test 対策、品質ゲートを PR の流れに組み込める。
