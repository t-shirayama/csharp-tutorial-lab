# 非同期処理のテスト

## 目的

`async` / `await` を使う処理を xUnit で安全にテストし、例外、キャンセル、timeout を検証できるようにします。

## 前提

- [xUnit の基本](01_xUnitの基本.md) を読んでいる
- [非同期プログラミング async-await](../05_非同期と並行処理/02_async-await.md) を読んでいる
- [テストしやすい設計](04_テストしやすい設計.md) を読んでいる

## 要点

- xUnit の非同期テストは `async Task` を返します。`async void` は失敗を正しく扱えないため、テストでは使いません。
- 非同期処理は必ず `await` します。`await` し忘れると、テストが処理完了前に成功してしまったり、例外を取り逃がしたりします。
- 非同期例外は `Assert.ThrowsAsync<TException>` で検証します。同期の `Assert.Throws` とは使い分けます。
- `CancellationToken` を受け取る処理は、キャンセルされたときの挙動をテストします。長時間処理や外部 API 呼び出しでは重要です。
- timeout のテストで `Task.Delay` や `Thread.Sleep` に頼りすぎると flaky になりやすいです。`TimeProvider` や差し替え可能な clock、短い fake を使います。
- 非同期テストは並列実行の影響を受けることがあります。共有 DB、static state、固定ファイル名などを使う場合は独立性を意識します。

```csharp
public class ProductServiceTests
{
    [Fact]
    public async Task FindAsync_ReturnsProduct_WhenProductExists()
    {
        var repository = new InMemoryProductRepository();
        var service = new ProductService(repository);

        await repository.AddAsync(new Product(1, "Keyboard", 12000m));

        var product = await service.FindAsync(1);

        Assert.NotNull(product);
        Assert.Equal("Keyboard", product.Name);
    }
}
```

## 非同期例外を検証する

```csharp
[Fact]
public async Task CreateAsync_Throws_WhenNameIsEmpty()
{
    var service = new ProductService(new InMemoryProductRepository());

    var exception = await Assert.ThrowsAsync<ArgumentException>(() =>
        service.CreateAsync("", 12000m));

    Assert.Equal("name", exception.ParamName);
}
```

## CancellationToken を検証する

```csharp
[Fact]
public async Task ImportAsync_ThrowsOperationCanceled_WhenCanceled()
{
    var service = new ImportService();
    using var cts = new CancellationTokenSource();

    cts.Cancel();

    await Assert.ThrowsAsync<OperationCanceledException>(() =>
        service.ImportAsync(cts.Token));
}
```

## 待ち忘れの悪い例

```csharp
[Fact]
public void BadTest_DoesNotWaitAsyncOperation()
{
    var service = new ProductService(new InMemoryProductRepository());

    // この呼び出しは await されていないため、例外や失敗を取り逃がす可能性があります。
    service.CreateAsync("", 12000m);
}
```

## コードの読み方

非同期テストでは、Arrange で repository や service を準備し、Act で `await service.FindAsync(...)` のように非同期処理を待ち、Assert で結果を確認します。例外テストでは `Assert.ThrowsAsync` 自体を `await` しています。`CancellationTokenSource` はキャンセルを発生させるための object で、`cts.Cancel()` の後に処理を呼ぶことでキャンセル済みの経路を確認しています。

## 実務での使い方

DB、外部 API、file I/O、message queue などは非同期 API になることが多いです。正常系だけでなく、not found、timeout、キャンセル、外部 API 失敗、リトライ不可の失敗をテストします。テストが遅くなる場合は、unit test と integration test を分けます。

## よくあるミス

- テストメソッドを `async void` にする。
- `Task` を返すメソッドを `await` しない。
- `Assert.Throws` で非同期例外を検証しようとする。
- `Thread.Sleep` で待機し、環境によって失敗するテストにする。
- キャンセル token を受け取っているのに、キャンセル時の挙動を確認しない。

## 練習問題

1. `async Task` の test method を作る。
2. 非同期 method の例外を `Assert.ThrowsAsync` で確認する。
3. `CancellationToken` がキャンセル済みのときの動作をテストする。
4. `await` を外した悪いテストがなぜ危険か説明する。
