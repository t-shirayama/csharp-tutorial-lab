# 静的解析

## 目的

実行前にコードの問題を検出する静的解析の基本を理解します。

## 要点

- compiler warning、analyzer、nullable warning は品質シグナルです。
- 警告を無視せず、理由がある場合だけ抑制します。
- ルールは厳しくしすぎると運用されなくなります。

## 実務での使い方

.NET analyzers、StyleCop、Sonar などを使います。まず nullable と compiler warning を綺麗に保つだけでも効果があります。

CI で `dotnet format --verify-no-changes` や analyzer を品質ゲートに組み込む例は [品質ゲートと PR チェック](18_品質ゲートとPRチェック.md) で扱います。

## よくあるミス

- warning を大量に放置する。
- `#pragma` で理由なく抑制する。
- ルールを増やしすぎて開発速度を落とす。
