# 集合、キュー、スタック

## 目的

`HashSet<T>`、`Queue<T>`、`Stack<T>` の使いどころを理解します。

## 前提

- [配列と List](01_配列とList.md) を読んでいる
- [Dictionary](02_Dictionary.md) を読んでいる

## 要点

- `HashSet<T>` は重複しない集合です。
- `Queue<T>` は先に入れたものを先に取り出す FIFO です。
- `Stack<T>` は後に入れたものを先に取り出す LIFO です。

## コード例

```csharp
// この例では「集合、キュー、スタック」の要点を最小のコードで確認します。
var tags = new HashSet<string>(StringComparer.OrdinalIgnoreCase);
tags.Add("CSharp");
tags.Add("csharp");

Console.WriteLine(tags.Count); // 1

var queue = new Queue<string>();
queue.Enqueue("first");
queue.Enqueue("second");
Console.WriteLine(queue.Dequeue()); // first

var stack = new Stack<string>();
stack.Push("first");
stack.Push("second");
Console.WriteLine(stack.Pop()); // second
```

## コードの読み方

`HashSet<string>` は重複排除に使えます。`Queue<T>` は処理待ち、`Stack<T>` は戻る操作や深さ優先探索のような場面で使います。

## 実務での使い方

重複チェック、未処理ジョブ、探索処理、undo 履歴などで使います。複数 thread から同時に使う場合は、通常の collection ではなく thread-safe collection や channel を検討します。

## よくあるミス

- `List<T>.Contains` で大量データの重複チェックを行う。
- `Queue<T>` と `Stack<T>` の取り出し順を混同する。
- 空の queue / stack から取り出して例外にする。

## 練習問題

1. `HashSet<string>` で重複する ID を除外する。
2. `Queue<string>` で処理待ち一覧を作る。
3. `Stack<string>` で簡単な undo 履歴を表す。
